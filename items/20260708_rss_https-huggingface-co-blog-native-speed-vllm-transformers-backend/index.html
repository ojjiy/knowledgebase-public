Native-speed vLLM transformers modeling backend

Date: 2026-07-08

Source: rss / Hugging Face Blog

Original URL

vLLMのトランスフォーマーモデリングバックエンドがネイティブ速度で利用可能になり、多くのLLMアーキテクチャにおいて、カスタムvLLM実装と同等またはそれ以上の推論速度を実現しました。これにより、モデル作者は追加のコード移植なしに、自身のトランスフォーマー実装をそのまま利用して超高速なvLLM推論が可能になります。 この機能は、Hugging FaceのトランスフォーマーライブラリとvLLMを統合することで実現されました。トランスフォーマーがモデリングコードを提供し、vLLMが連続バッチ処理やカスタムアテンションカーネルなどの最適化された推論技術を提供します。 今回のアップデートでは、動的な推論特化型レイヤー融合が実行時に適用され、カスタムコード実装と同等の速度を達成できるようになりました。これは、torch.fxによる静的解析で最適化可能なパターンを検出し、ast（抽象構文木）を用いてソースコードを書き換えることで実現されています。これにより、Expert Parallelization（EP）やTensor Parallelism（TP）、Pipeline Parallelism（PP）のための最適化されたvLLMカーネルが利用可能になります。 この変更により、モデル作者は追加の最適化コードを書く必要なく、ネイティブvLLM実装と同等の推論速度を得ることができます。さらに、トランスフォーマー実装は学習にも使用できるため、学習、評価、RLロールアウトで同じモデルコードを共有できます。 利用するには、vLLMパッケージをアップグレードし、`--model-impl transformers` フラグを指定するだけで、既存のサービング設定を変更することなく利用できます。ただし、線形アテンションを使用するモデルや、Hubリポジトリでカスタムコードが管理されているモデルは、現時点ではサポートされていません。 今後、この最適化手法の詳細について、さらに詳しいブログ記事が公開される予定です。
