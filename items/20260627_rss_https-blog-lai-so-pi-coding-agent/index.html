作って使うAIエージェント —— Pi Coding Agentで足りない機能を自作しよう

Date: 2026-06-27

Source: rss / Hatena Bookmark LLM

Original URL

「Pi Coding Agent」は、Mario Zechner氏が開発するOSSのターミナル型コーディングエージェントです。近年のAIエージェント開発では、モデル自体の性能だけでなく、継続実行ループ、サブエージェント、メモリ、外部ツールなどをどのように組み合わせるか、という「ハーネス層」の重要性が増しています。これは、最先端モデルの提供が段階的になったり、アクセス制限が課されたりする状況も背景にあります。 Pi Coding Agentは、「最初から全部入り」の機能的なエージェントではなく、TypeScriptの拡張機能（extension）によってユーザーが不足している機能を自作し、カスタマイズしていくことを前提とした「minimal terminal coding harness（最小限のターミナルコーディングハーネス）」として位置づけられています。 **Pi Coding Agentの主な特徴と利点:** * **拡張性の高さ:** TypeScriptのextension APIを通じて、スラッシュコマンド、LLMが呼び出すツール（tool）、実行中のイベントに反応するフック（hook）、UI表示、モデルプロバイダーなどを追加できます。これにより、一般的なコーディングエージェントよりもさらに低レイヤーのランタイム部分を拡張できます。 * **抽象レイヤーの薄さ:** LLM APIに最小限のツールを渡し、その結果をそのまま扱うシンプルな構成のため、新しいモデルやプロバイダーを試す際に、モデル自体の挙動を観察しやすいという利点があります。OpenAI Chat Completions互換APIなどを採用しているモデルであれば、provider設定を追加するだけで容易に試すことができます。 * **「作って使う」思想:** 既存の機能を追加するだけでなく、はてなブックマーク検索、Web検索、X検索、メモリレイヤー（cman）、Tailscale内のローカルLLM連携などの機能を、extensionや外部CLIとして自作し、自分のワークフローに合わせて組み込むことができます。これにより、汎用エージェントから自分専用のハーネスへと進化させることができます。 * **MCPやSkillsとの違い:** SkillsがLLMに渡す情報を増やすのに向いているのに対し、PiのextensionはPiのプロセス内でTypeScriptモジュールとして読み込まれるため、toolの追加だけでなく、実行途中の処理の挟み込み（hook）やUI更新、プロバイダー追加など、より広範な拡張が可能です。 * **セキュリティへの注意:** extensionはTypeScriptコードとして実行されるため、信頼できないextensionやPi packageの導入は、ローカルファイル、シェル、認証情報へのアクセスにつながる可能性があり、注意が必要です。APIキーの扱いやログに記録する内容、外部APIに送る情報についても、自己責任での管理が求められます。 **具体的な拡張例:** * **はてなブックマーク検索:** 日本語圏のコミュニティの反応やまとめ記事を取得するために、はてなブックマーク検索のRSSを利用したextensionを作成。 * **Web検索:** Brave Search API、Serper、Tavily、Exa、SearXNGなど、自分で選んだ検索バックエンドをextensionとして差し込める。 * **X検索:** xAIの`x_search`ツールをAPIキー方式で利用するextensionを作成。 * **メモリレイヤー:** 過去のセッションログを検索して作業を思い出させる`cman`ツールをPi packageとして追加。 * **Tailscale連携:** 自宅のGPUマシンで動かすローカルLLMとの連携を、extensionや共有ファイル参照といった形で実現。 Pi Coding Agentは、単なる高機能なコーディングエージェントではなく、AI機能をユーザー自身が自由に開発・統合するための柔軟な基盤（ハーネス）として、モデル単体の性能だけでなく、それらをどのように組み合わせるかという実験を容易にするツールと言えます。
