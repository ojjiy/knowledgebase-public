# タイトル Infinite Worlds with Versatile Interactions [View Paper](http://arxiv.org/abs/2607.07534v1) ## 1. 問題意識：解決したい問題はなにか 本論文では、生成AIを用いて無限に広がるインタラクティブな世界をリアルタイムで生成するという目標に、既存の技術では達成が困難な2つの主要な課題が存在すると指摘しています。 1. **長期的な安定性の欠如（エラーの蓄積）**: 生成モデルがフレームを逐次生成する際に、以前のフレームで発生したエラーが次のフレーム生成にフィードバックされ、時間とともにテクスチャの劣化、ジオメトリの歪み、シーンの漂流といった現象が生じ、生成される世界が短時間しか安定しないという問題です。これは、期待される「住むことができる」ような長期間持続する世界とはかけ離れた状態を引き起こします。 2. **インタラクティビティの低さ（計算コスト）**: 高解像度で滑らかなビデオを生成しながらリアルタイムな入力に応答することは計算コストが非常に高く、従来の手法では解像度、滑らかさ、あるいは制御性を犠牲にする必要がありました。その結果、ユーザーは粗いカメラ操作しかできず、インタラクティブ性が著しく損なわれるという問題です。 これらの課題により、真にオープンエンドで探索可能なインタラクティブな世界を生成することが困難になっていました。 ## 2. 提案手法のアプローチ：どのようなアイデア・アプローチで問題を解決しようとしたか 本論文では、上記の2つの課題を直接的に解決するため、以下の2段階のアプローチを採用しています。 1. **因果的（Causal）な事前学習による長期安定性の向上**: * エラーの蓄積に抵抗力を持つように設計された因果的ビデオ生成モデルを学習させます。これにより、生成される世界の質が長時間持続し、後続の蒸留プロセスのための高品質な教師モデル（teacher model）となります。 * 「Mixture of Bidirectional and Autoregressive (MoBA) Attention Mask」を導入し、教師強制（teacher forcing）による過学習と品質低下を防ぎつつ、双方向（bidirectional）コンポーネントを組み込むことで、柔軟な長さのビデオ生成への適応と、双方向から自動回帰（autoregressive）生成への遷移を促進します。 2. **蒸留（Distillation）によるリアルタイム性とインタラクティビティの実現**: * 事前学習で得られた高品質だが計算コストの高いモデルを、リアルタイム生成に適した軽量なモデル（student model）に蒸留します。 * 「Consistency Distillation (CD)」により、複数ステップの教師モデルのデノイズ軌跡を数ステップの生成器に圧縮し、サンプリングコストを削減します。 * 「Distribution Matching Distillation (DMD)」により、数ステップの生成器の視覚的忠実度や長期的なドリフトをさらに改善し、自己ロールアウト（self-rollout）での安定性を高めます。 * 「Agentic Interaction Harness」を導入し、世界モデリングのドメインにパイロットエージェント（キャラクターの行動計画・実行）とディレクターエージェント（環境要素の合成）を組み込むことで、単なるビデオ生成から、自律的で目標指向性のあるインタラクティブな体験へと昇華させます。 * 「Director-Pilot」アーキテクチャを採用し、VLM（Vision-Language Model）を「ディレクター」として、ビデオ生成モデル（Diffusion Transformer）を「パイロット」として機能させることで、意味論的なルールと物理的なダイナミクスの両方を処理します。 これらのアプローチを組み合わせることで、長期安定性とリアルタイム性を両立させ、多様なインタラクションが可能な無限の世界を生成することを目指しています。 ## 3. 成果：提案手法により、何が達成できたのか 本論文の提案手法により、以下の顕著な成果が達成されました。 * **無制限のインタラクションホライゾンと一貫した品質**: 厳密に設計された因果的事前学習パラダイムにより、モデルは出力品質を維持しながら、無限のインタラクションホライゾン（生成される世界の広がり）を達成しました。1時間以上の連続生成セッションでも品質の低下が見られないことが実証されました。 * **リアルタイム性能の実現**: ベースモデルからリアルタイム版を蒸留することにより、60fpsの720pビデオストリームを駆動するのに十分な応答速度を実現しました。 * **多様なインタラクティブ要素の導入**: 以前のバージョンと比較して、攻撃、射撃、呪文詠唱など、より広範なアクションと、テキスト駆動型のイベントが豊富になり、インタラクティブ性が大幅に向上しました。 * **ワールドモデリングにおけるエージェントハーネスの先駆的統合**: * **パイロットエージェント**: キャラクターの行動計画と実行を担当します。 * **ディレクターエージェント**: シーンの進行に合わせて新しい環境要素を合成する責任を負います。 これにより、自律的で、目標指向性のある、オープンエンドな体験が実現しました。 * **複数プレイヤーによる共有体験の促進**: 複数のプレイヤーが同時にこの鮮やかな世界シミュレーターに没入できるインターフェースが開発されました。 * **効率的なデプロイメント**: 主力となる14Bモデルと、単一GPUでの容易なデプロイメントをサポートする軽量な1.3Bモデルがペアリングされています。 * **最先端のオープンソースモデル**: 1時間レベル（無限）の生成持続時間を達成し、高い動的自由度、意味論的インタラクション、リアルタイム性能を組み合わせた唯一のモデルとして、比較対象のモデルを凌駕しました。 これらの成果により、過去のモデルでは困難であった、長期間安定し、リアルタイムで多様なインタラクションが可能な、無限に広がる世界を生成することが可能になりました。 ## 4. 課題：本手法の限界・課題にはなにか。本文中で著者が言及しているものと、あなたが想定するもの 本文中で著者が言及している限界・課題は以下の通りです。 * **真の記憶機能の欠如**: モデルは視覚的に安定した世界を生成しますが、生成された世界を「記憶」しているわけではありません。コンテキストウィンドウから外れた領域を後で再訪した場合、記憶から呼び出されるのではなく、再生成される傾向があります。これは、外見は永続的でも、アイデンティティ（同一性）は永続的ではないことを意味します。動的なKVキャッシュスケジューリングのようなメカニズムが探索されていますが、メモリコストと予算の間のトレードオフが存在します。 * **長期的なアートスタイルのドリフト**: 長期的なロールアウト（生成の連続）において、特定のキャラクターの外観が微妙に変化したり、全体的なアートスタイルが徐々にドリフトしたりする可能性があります。 * **物理的相互作用の不完全さ**: モデルはピクセルからのみダイナミクスを学習し、ジオメトリや衝突の明示的な概念を持っていません。その結果、物理的にありえないアーティファクト（例: キャラクターやオブジェクトが衝突せずに通り抜ける）が時折発生する可能性があります。 * **計算リソースの要求**: 提示されたモデルはリアルタイムで動作しますが、依然として substantial な計算リソースを必要とします。コモディティハードウェア上でリアルタイムかつ高忠実度な世界モデリングを実現するには、さらなる効率化が必要です。 私が想定する追加の課題としては、以下の点が考えられます。 * **生成されるコンテンツの多様性と創造性**: 現在のモデルは、既存のデータセットから学習したパターンに基づいてコンテンツを生成します。真に斬新で予測不可能な、あるいは極めて創造的なコンテンツを生成する能力には限界がある可能性があります。 * **倫理的・安全性の考慮**: 生成される世界が、不適切または有害なコンテンツ（暴力、差別、誤情報など）を生成しないようにするための、より高度なガードレールやコンテンツモデレーションの仕組みが必要となる可能性があります。 * **ユーザーインターフェースの複雑さ**: エージェント制御サーフェスには多数の「イベント提案」が表示され、ホットキーに割り当てられています。ユーザーがこれらの多数のオプションを効果的に活用し、意図した通りのインタラクションを行うためには、熟練が必要となる場合があります。より直感的で、ユーザーの意図を汲み取るインターフェースの改善が望まれます。 * **データバイアスの影響**: 学習データに存在するバイアスが、生成される世界やキャラクターの表現に影響を与える可能性があります。 ## 5. 技術詳細：提案手法の技術的な詳細について、できるだけ詳しく。数式やアルゴリズム、モデル構造なども可能な限り詳しく 本論文で提案されている技術的な詳細について、以下に詳しく説明します。 ### 1. モデルアーキテクチャと全体像 * **LingBot-World 2.0 (LingBot-World-Infinity)**: 4つの主要なアップグレードを持つ高度な反復モデルです。 * **無制限のインタラクションホライゾン**: 因果的（autoregressive）な事前学習パラダイムにより、品質を維持しながら無制限のインタラクションホライゾンを実現。 * **リアルタイム性能**: ベースモデルからリアルタイム変種を蒸留し、60fpsの720pビデオストリームを駆動する応答速度を保証。 * **多様なインタラクティブ要素**: 攻撃、射撃、呪文詠唱などのアクションと、豊かなテキスト駆動型イベント。 * **エージェントハーネスの統合**: パイロットエージェント（行動計画・実行）とディレクターエージェント（環境要素合成）によるワールドモデリング。 * **14Bモデルと1.3Bモデル**: 主力モデルと軽量モデルのペアリングにより、GPU1枚でのデプロイメントを容易に。 * **Director-Pilot アーキテクチャ**: * **ディレクター (VLM)**: マクロな意味論的ルールと因果関係を管理。現在の視覚的現象を分析し、ユーザーインタラクションの論理的結果を予測し、「イベント提案」を生成。 * **パイロット (Diffusion インタラクティブなワールドモデルをトレーニングするための、時間変化する指示や制御信号に応答するデータセットを構築。 * **3段階**: 1. **メタデータスキーマの正規化**: 異種ソースからのビデオを共有スキーマに正規化。 2. **フィルタリングとVLMプロファイリング**: 低品質または意味論的に不適切なサンプルを除去。 3. **構造化アノテーション**: グローバルキャプション、チャンクごとのローカルキャプションを生成。 * **データソース**: 自己収集したエゴセントリックビデオ、ゲームやUnreal Engineからの合成データ、大規模ウェブビデオ。 * **フィルタリング**: * **技術的フィルタリング**: 動画デコーディングの検証、無効サンプルの除去、ショット境界検出。TransNet V2 によるクリップ分割。 * **スコアリングフィルタ**: 視覚品質、輝度範囲、OCRテキスト占有率、モーション統計、エンコーディング安定性などの指標を統合。 * **VLMプロファイリング**: サンプル品質（編集アーティファクト、曖昧さ）、意味論的コンテンツ（視点、アクティビティ、インタラクションパターン）を属性化。 ### 3. 因果的ワールドモデリング (Pre-training Stage) * **目的**: 無制限で、アクション制御可能、高忠実度のビデオワールドを生成する因果的ワールドモデルを学習。 * **因果的生成プロセス**: ``` # 潜在状態のシーケンス: x_{1:T} # ユーザー入力のシーケンス: a_{1:T} # モデルパラメータ: theta # causal assumption: p_theta(x_{1:T} | a_{1:T}) = product_{t} p_theta(x_t | x_{<t}, a_{<=t}) ``` * **2つのユーザー入力 (アクション)**: * **カメラポーズ**: Plücker埋め込みで表現され、Adaptive Layer Normalization (AdaLN) を通じて拡散プロセスに組み込まれる。 * **テキストプロンプト**: オートリグレッションビデオ生成のためのチャンクごとのプロンプト。 * **Mixture of Bidirectional and Autoregressive (MoBA) Attention Mask**: * **自己アテンション**: * 教師強制（Teacher forcing）マスク: 現在のノイズ付きフレーム `x_i^t` が、自身とクリーンなコンテキスト `x_{<i}` のみにアテンション。 * 双方向コンポーネント: 全てのフレームにフルアテンション。これにより、可変長ビデオ生成への適応と、双方向から自動回帰への遷移を促進。教師強制の過学習を緩和する正則化器としても機能。 * **クロスアテンション**: * 教師強制コンポーネント: バックグラウンドプロンプト `p_bg` に、現在のチャンクおよびそれ以前の全てのチャンク `p_{<=i}` に対して、下三角パターンでアテンション（未来情報の漏洩を防ぐ）。 * 双方向コンポーネント: グローバルプロンプト `p_global` にアテンション。 * **学習目標**: Conditional flow-matching (CFM) オブジェクティブ。 ``` # 目的関数: L_fm = E[ || v_theta(x_i^t, t | x_{<i}, p_{<=i}, a_{<=i}) - (epsilon - x_i) ||^2 ] # v_theta: モデルが予測するフロー速度 # x_i^t: 時刻tでのノイズ付き状態 # epsilon: ノイズ # x_i: クリーンな状態 # p_{<=i}: 過去のプロンプト # a_{<=i}: 過去のアクション（カメラポーズ、プロンプト） ``` ### 4. 蒸留とリアルタイム化 (Post-training Stage) * **目的**: 事前学習された多ステップの因果的拡散ワールドモデルを、リアルタイムインタラクションに適した数ステップのジェネレーターに圧縮し、長期ロールアウトでのエラー蓄積を抑制。 * **Consistency Distillation (CD)**: * 教師モデルのPF-ODE (Probability Flow Ordinary Differential Equation) 軌跡上の潜在状態が、同一の予測にマッピングされることを強制。 * 目的関数: `L_CD = E[ d(G_theta(x_i^t, t | c), G_{theta^-}(x_i^{t-dt}, t-dt | c)) ]` * `G_theta`: 学生（student）モデル * `theta^-`: 学生パラメータの指数移動平均 (EMA) * `c`: コンテキスト * `d`: 距離関数 * **Distribution Matching Distillation (DMD)**: * 学生ジェネレーターの分布とデータ分布のKLダイバージェンスの勾配を最適化。 * 目的関数勾配: `∇_theta E[D_KL(p_theta,t || p_data,t)] = -E[ (s_real(hat{x}_i^t, t | c) - s_fake(hat{x}_i^t, t | c)) * d(hat{x}_i)/d(theta) ]` * `s_real`: ノイズ付きデータ分布のスコア近似 * `s_fake`: ノイズ付き学生分布のスコア近似 * `hat{x}_i`: 学生生成サンプル * 長期自己ロールアウト軌跡全体にDMDを適用し、累積ドリフトを削減。 ### 5. インタラクションハーネスとインターフェース * **Agentic Interaction Harness**: * **Director (VLM)**: 現在のフレームとユーザーアクションを分析し、論理的なイベント提案を生成。 * **Pilot (DiT)**: イベント提案を物理的に一貫したビデオロールアウトにグラウンド。 * **インタラクションモード**: * **モードA (セマンティック/オブジェクト非依存)**:
