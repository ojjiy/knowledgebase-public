Amazon Bedrock 上の OpenAI モデル（GPT-5.5）をプロダクトに組み込んでみて得た知見

Date: 2026-06-27

Source: rss / DevelopersIO

Original URL

Amazon Bedrock 上の OpenAI モデル（GPT-5.5）をプロダクトに組み込んだ際の知見が共有されています。 **主なポイント:** * **クライアント作成:** OpenAI SDK の `BedrockOpenAI` クライアントを使用し、`@aws/bedrock-token-generator` を介して AWS クレデンシャルから Bedrock 用のベアラートークンを都度生成します。これにより、OpenAI プラットフォーム用の API キーは不要になり、`awsRegion` を指定することで `baseURL` も自動的に導出されます。 * **ストリームの delta セマンティクス:** 純正 OpenAI では増分（差分）として届く delta が、Bedrock 経由では累積スナップショット（全文の再送）と混在して届く場合があります。これをそのまま連結すると文章が重複するため、現在保持しているテキストと新規に届いた delta を比較し、累積スナップショットの場合は全文で置き換え、増分 delta の場合は追記するという専用の変換処理が必要になります。 * **その他の制約:** * PDF 入力は、`file_data` に `data:application/pdf;base64,<base64文字列>` 形式で base64 エンコードされたデータをインライン埋め込むことで可能です。 * `hosted tools`（web_search, file_search, code_interpreter）は Bedrock エンドポイントではサポートされていません。 * データの取り扱いは他の Bedrock モデルと異なるため、`abuse detection` のドキュメントを確認し、ユーザーへの注意喚起が必要です。 **結論として、** Amazon Bedrock 上の OpenAI モデルは、AWS クレデンシャルで OpenAI 互換エンドポイントを呼び出す形であり、基本的な部分の構築はスムーズに進められます。しかし、ストリームの delta の挙動が純正 OpenAI と異なるため、そこへの対応が実装上の主要な課題となります。
