Probelock – lockfile for LLM tool calling

Date: 2026-07-09

Source: rss / Hacker News LLM

Original URL

Probelockは、LLM（大規模言語モデル）のツール呼び出し能力をテスト・監視するためのロッカーファイル（lockfile）およびテストフレームワークです。モデルのツール呼び出しに関する能力を記録し、モデルの変更（例：量子化レベルの変更、ランタイムの変更）によって能力が低下した場合にCI（継続的インテグレーション）を失敗させることができます。 Probelockの主な特徴は以下の通りです。 * **能力の記録と監視**: モデルがツール呼び出しと出力チェックにおいてどのような能力を発揮するかを記録します。 * **CIでの回帰検出**: モデルの変更が能力の低下につながった場合、CIパイプラインで自動的に検出・通知します。 * **テストケース不要**: プロンプトエンジニアリングの専門知識がなくても、ツールのスキーマ定義から自動的にテスト（プローブ）を生成します。 * **コードによるスコアリング**: LLMによる評価ではなく、JSONスキーマ検証、完全一致、ツール名チェックなどのコードベースで客観的にスコアリングします。 * **再現性**: 同じモデルに対して実行すれば、常に同じ結果が得られます。 * **比較ベース**: 絶対的なリーダーボードではなく、モデル自身のベースラインと比較することで、モデルの変更による相対的な影響を評価します。 * **多様なプロバイダー・フレームワーク対応**: OpenAI互換APIを持つもの（Ollama, vLLM, Llama.cpp Server, LM Studio, HF TGIなど）や、Anthropic, GeminiなどのプロバイダーもSDK経由で利用可能です。 * **リアルデータからのプローブ生成**: * **トレースからのプローブ生成 (`--traces`)**: 実際の対話履歴から、より現実的なテストケースを生成します。 * **ログからのマイニング (`probelock ingest`)**: エージェントの生のログから、最小限の労力で多段階のテストケースをマイニングします。 * **トレンド分析**: 複数のロックファイルを比較し、モデルの能力の経時的な変化や、量子化レベルごとの能力の推移を分析できます。 * **サンプリングとノイズゲート**: サンプリング数を増やし、統計的な有意性に基づいてゲートを失敗させることで、ノイズの多い結果による誤検知を防ぎます。 Probelockは、`uvx probelock --help` で最新版を実行したり、`pip install probelock` でインストールして利用できます。CLIコマンドを通じて、プローブの生成、実行、比較、ゲート処理などが行えます。 総じて、ProbelockはLLMのツール呼び出し能力の安定性と回帰を保証するための、開発者フレンドリーで客観的な評価・監視ツールと言えます。
