# タイトル Flex-Forcing: Towards a Unified Autoregressive and Bidirectional Video Diffusion Model [View Paper](http://arxiv.org/abs/2607.03509v1) ## 1. 問題意識：解決したい問題はなにか 既存のビデオ生成モデルは、推論のパラダイムが固定されているという制約がありました。具体的には、以下の2つの主要なアプローチがそれぞれ長所と短所を持っていました。 * **双方向拡散モデル (Bidirectional diffusion models):** 全体的な整合性や視覚的な忠実度に優れますが、推論が遅いという欠点がありました。これは、グローバルな構造計画には適していますが、リアルタイム生成やストリーミング生成には不向きでした。 * **自己回帰モデル (Autoregressive models):** 効率的でストリーミング生成が可能ですが、長距離の整合性の欠如や露出バイアス（exposure bias）といった問題がありました。これは、リアルタイム性や長尺動画生成には適していますが、動画全体の首尾一貫性を保つのが難しい場合がありました。 これらの既存手法は、片方の利点を追求するともう片方の利点を犠牲にする必要がありました。そこで、これらの補完的な利点を単一のモデルで統一的に実現し、高品質なビデオ生成と効率的な長尺ビデオ生成を両立させるための、より柔軟な推論フレームワークが必要とされていました。 ## 2. 提案手法のアプローチ：どのようなアイデア・アプローチで問題を解決しようとしたか 提案手法である「Flex-Forcing」は、単一のビデオ拡散モデルが双方向生成と自己回帰生成の両方のモードでシームレスに動作できるようにする、統一的な学習および推論フレームワークを提案しています。その核心的なアイデアは、**時間軸とノイズ除去ステップの両方で定義される柔軟なチャンキングメカニズム (flexible chunking mechanism)** です。 このメカニズムにより、モデルは以下のことが可能になります。 1. **デバイス予算に応じた柔軟なチャンキング:** 異なる計算リソース（デバイス予算）に合わせて、チャンクの分割方法を調整できます。 2. **双方向推論と自己回帰生成の組み合わせ:** チャンク間では双方向推論を用いてグローバルな構造計画を行い、各チャンク内では自己回帰的にフレームを生成することで、効率的かつきめ細やかな合成を実現します。 3. **任意の順序・任意のタイムステップでの自己回帰生成:** 厳密な因果関係の制約なしに、任意の順序で任意のタイムステップの自己回帰生成を実行できます。 この柔軟なチャンキングは、時間軸（ビデオフレーム）とノイズ除去ステップ（デノイズングタイムステップ）の2つの直交する軸で定義されます。これにより、モデルは多様な因果関係と非因果関係のコンテキストにさらされ、双方向生成と自己回帰生成の両方の能力を学習します。さらに、異なるノイズレベルを持つコンテキスト間の表現の一貫性を強制する**ノイズアラインメントメカニズム (noise-aligned projection, K-Projection)** を導入し、パフォーマンスの安定化を図っています。 ## 3. 成果：提案手法により、何が達成できたのか Flex-Forcingを導入することで、以下の顕著な成果が達成されました。 * **品質と効率のトレードオフの改善:** 従来の固定された推論スケジュールを持つベースライン手法と比較して、一貫して優れたビデオ品質と長尺ビデオの安定性を達成しました。同時に、推論速度も向上させました。これは、品質と効率の間の「パレートフロンティア」を大幅に改善したことを意味します。 * **単一モデルでの柔軟な推論:** 単一の学習済みモデルで、高品質な双方向推論、効率的な長尺ビデオ・ストリーミング生成のための自己回帰推論、およびその中間的なトレードオフを実現する半自己回帰ハイブリッドモードを切り替えることが可能になりました。 * **高速な推論:** 特定の構成では、従来の双方向モデルよりも大幅に高速な推論を実現しました。例えば、5秒のビデオ生成において、FPS (frames per second) が向上しました。 * **長尺ビデオ生成の安定性向上:** 従来の自己回帰モデルで問題となっていた、オブジェクトの見た目、動き、シーン構造のドリフトや、グローバルな時間的整合性の弱さといった問題が軽減され、長尺ビデオの安定性が向上しました。 * **新しい編集能力の実現:** 「任意の順序、任意のタイムステップでの自己回帰編集 (any-order, any-timestep autoregressive editing)」といった、従来の自己回帰モデルでは困難であった、局所的な時間的編集を行いながらグローバルなビデオ整合性を維持する能力が向上しました。 * **計算リソースの効率化:** 異なる推論モードのために複数のモデルを維持・管理する必要がなくなり、計算コストと環境負荷を削減しました。 ## 4. Flex-Forcingの技術的な詳細を以下に説明します。 ### 1. 柔軟なチャンキングメカニズム (Flexible Chunking Mechanism) Flex-Forcingの核となるのは、時間軸（ビデオフレーム）とノイズ除去ステップ（デノイズングタイムステップ）の2つの直交する軸で定義される柔軟なチャンキング戦略です。 **a. 時間軸における柔軟なチャンキング:** ビデオフレームのインデックス集合 $\mathcal{F} = \{0, 1, \ldots, F-1\}$ を、連続したフレームのチャンクの集合 $\{\mathcal{F}_{t,k}\}$ に分割します。ここで、 $\mathcal{F}_{t,k}$ はタイムステップ $t$ における $k$ 番目のチャンクを表し、そのフレームインデックスは $[a_{t,k-1}, a_{t,k})$ の範囲にあります。$a_t = (a_{t,0}, a_{t,1}, \ldots, a_{t,K_t})$ は、タイムステップ $t$ における境界設定（フレームの区切り点）です。 この分割により、チャンクのサイズは異なり得ます。 * **双方向推論:** 全てのフレームが互いにアテンションできる状態。 * **自己回帰推論:** チャンク $\mathcal{F}_{t,k}$ は、それ以前のチャンク（$\mathcal{F}_{t,<k} = \bigcup_{u<k} \mathcal{F}_{t,u}$）の推論結果のみに依存します。 `x_t-1^F_t,k ~ q_theta(x_t-1^F_t,k | x_0^F_t,<k, x_t^F_t,k; a_t)` * **ハイブリッド推論:** チャンク内では双方向アテンション、チャンク間では自己回帰的な生成を行います。 **b. ノイズ除去ステップにおける柔軟なチャンキング (Pyramid Chunking):** チャンクの分割方法 $\mathbf{a}_t$ は、ノイズ除去ステップ $t$ に依存して変化させることができます。 $t$ ステップ目のデノイズングが完了した後、$t+1$ ステップ目では、既存のチャンク境界をさらに細分化することができます。例えば、タイムステップ $t$ でのチャンク $\mathcal{F}_{t,k}=[a_{t,k-1}, a_{t,k})$ が、タイムステップ $t+1$ では新しい分割点 $\mathcal{S}_{t,k}$ によって $\mathcal{F}_{t+1,k}$ のように細分化されることがあります。 $\mathbf{a}'_{t,k} = a_{t,k-1} \cup \mathcal{S}_{t,k} \cup a_{t,k}$ この「ピラミッドチャンキング」により、ノイズが多い初期のデノイズングステップでは大きなチャンク（グローバル構造の把握に有利）を使用し、ノイズが少ない後期のステップでは小さなチャンク（局所的な詳細の生成に有利）を使用するといった階層的な処理が可能になります。 推論時には、大きなチャンクが細分化される場合、一時的なバッファリングとKVキャッシュの利用により、因果関係を維持しながら処理が継続されます。 ### 2. 学習目標 モデルは、双方向生成能力を維持しつつ、自己回帰生成能力を注入するように学習されます。これは、主に「非対称蒸留 (asymmetric distillation)」の段階で行われます。 * **因果制約の注入と非因果能力の維持:** CausVidなどの手法を参考に、因果関係を注入するために、学習時にランダムなチャンク分割 $\mathbf{a}_t$ をサンプリングし、この柔軟なチャンクに対してロールアウト（推論プロセス）を実行します。これにより、モデルは厳密な因果関係から完全な非因果関係までの様々なアテンションパターンにさらされます。 * **拡散モデルの損失関数:** 基本的には、拡散モデルの標準的な損失関数（例: $L_2$ 損失やVSD損失）が使用されます。 `∇_theta D_KL = E[-(s_real(x_t) - s_fake(x_t)) * ∂G_theta/∂theta]` ここで、$s$ はスコア関数、$G_theta$ はノイズ除去モデル、$x_t$ はタイムステップ $t$ におけるノイズ付きデータです。 ### 3. ノイズアラインメントメカニズム (Noise-Aligned Projection / K-Projection) 異なるノイズレベルを持つコンテキスト間でのアテンションのパフォーマンスを安定させるためのメカニズムです。 * **問題点:** 自己回帰的な生成（過去のチャンク）は、よりノイズ除去された状態（cleaner representations）から生成されることが多い一方、未来のチャンク（非因果コンテキスト）は、よりノイズの多い状態（noisier latents）から生成されることがあります。標準的なSelf-Attentionでは、これらの異なるノイズレベルを持つキー・バリューペアを均一に扱ってしまうため、パフォーマンスが低下します。 * **解決策 (K-Projection):** キー(K)の状態を、現在のデノイズングタイムステップ $t$ に対応する「ノイジー」な潜在空間に明示的に投影する線形変換 $\Pi_{t \leftarrow 0}$ を導入します。 `tilde K_t = Pi_{t <- 0}(K_0)` ここで、$K_0$ はクリーンな（ノイズ除去された）状態からのキー、$\tilde{K}_t$ はタイムステップ $t$ に適応されたキーです。 * **実装:** この投影は軽量な線形変換で、タイムステップに依存します。学習時や推論時には、アテンション計算の直前に、キャッシュされたKVペアに対してこの投影が適用されます。 `Attn(Q_t, tilde K_t, V_t) = softmax((Q_t * tilde K_t^T) / sqrt(d)) * V_t` ここで、$\tilde{K}_t = concat(\Pi_{t \leftarrow 0}(K_0^{<F_t,k}), \tilde{K}_t^{F_t,k})$ となります。 * **利点:** * 全てのキー・バリューペアがノイズ整合性の取れた表現空間で扱われるため、標準的なSelf-Attentionが安定して適用できます。 * KVキャッシュの効率性を維持しつつ、因果・非因果両方のコンテキストで安定した推論を可能にします。 * 学習時にアイデンティティマッピングとして初期化され、軽量な追加パラメータで学習できます。 ### 4. モデル構造 * **ベースモデル:** 基本的には、既存のビデオ拡散モデル（例: Wan2.1-T2V-1.3B）が使用されます。これらのモデルは、Transformerベースのアーキテクチャ（例: Diffusion Transformer - DiT）を採用し、潜在拡散モデル（Latent Diffusion Models - LDM）のフレームワークに基づいていることが多いです。 * **アテンションメカニズム:** Self-Attention (Multi-Head Self-Attention) が中心的な役割を果たします。Flex-Forcingは、このSelf-Attentionメカニズムに柔軟なチャンキングとK-Projectionを適用します。 * **学習パイプライン:** CausVidなどの既存手法を参考に、ODE初期化、非対称蒸留、DMDトレーニングなどの段階を経て学習が進められることがあります。 ### 5. 推論プロセス 1. **初期化:** 最初のデノイズングステップ $t$ では、ランダムノイズ $\mathbf{x}_t$ から開始します。 2. **チャンク処理:** * 時間軸とノイズ除去ステップの柔軟なチャンキング戦略 $\mathbf{a}_t$ に基づいて、現在のチャンク $\mathcal{F}_{t,k}$ が決定されます。 * チャンク内では、双方向アテンション（または因果制約下でのアテンション）が適用されます。 * K-Projectionが適用され、キー（K）の状態が現在のノイズレベルに整合されます。 * Self-Attentionが実行され、ノイズ除去された状態 $\mathbf{x}_{t-1}^{\mathcal{F}_{t,k}}$ が生成されます。 3. **KVキャッシュ:** 生成されたフレームのKVキャッシュが保存され、後続のチャンクやステップでの自己回帰的な推論に利用されます。 4. **ステップ更新:** デノイズングステップ $t$ が $t-1$ に更新され、ノイズ除去が繰り返されます。 5. **チャンク分割の動的変更:** ノイズ除去ステップが進むにつれて、ピラミッドチャンキングによりチャンクの分割がより細かくなる場合があります。この場合、必要に応じてKVキャッシュを適切に管理し、因果関係を維持しながら処理を進めます。 6. **最終生成:** デノイズングステップが完了すると、生成されたビデオ $\mathbf{x}_0$ が得られます。 このプロセスにより、ユーザーは推論時にチャンクの粒度や配置を調整することで、品質、速度、計算リソースの間のトレードオフを柔軟に制御できます。 ## 6. この論文を140字以内のツイートで要約すると？
