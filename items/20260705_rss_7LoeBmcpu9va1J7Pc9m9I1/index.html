Claude Codeの回答待ちに対する自動続行設定(askUserQuestionTimeout)について調査してみた

Date: 2026-07-05

Source: rss / DevelopersIO

Original URL

Claude Codeでは、エージェントが作業中にユーザーに選択式の質問を投げかける「AskUserQuestion」という機能があります。しかし、この質問が表示されたままユーザーが離席すると、回答があるまでエージェントの作業が停止してしまうという課題がありました。 この問題を解決するために、**`askUserQuestionTimeout`** という設定キーが導入されました。この設定を有効にすると、AskUserQuestionの質問ダイアログが一定時間操作されない（アイドル状態が続く）場合に、その時点で選択されているオプションを自動的に採用して処理を続行します。 **`askUserQuestionTimeout` の主な仕様:** * **設定キー:** `askUserQuestionTimeout` * **デフォルト値:** `"never"` (無期限に回答を待つ) * **指定可能な値:** `"60s"`、`"5m"`、`"10m"` のような時間指定文字列、または `"never"` * **設定スコープ:** ユーザー設定 (`~/.claude/settings.json`) のみ * **必要バージョン:** Claude Code v2.1.200以降 **設定方法:** ユーザー設定ファイル (`~/.claude/settings.json`) に、トップレベルキーとして `askUserQuestionTimeout` を追加します。例えば、5分間操作がなければ自動続行させたい場合は、以下のように記述します。 ```json { "askUserQuestionTimeout": "5m" } ``` **注意点:** * **ユーザースコープ専用:** この設定はユーザー個人に紐づく設定であり、プロジェクト設定 (`.claude/settings.json`) やローカル設定 (`.claude/settings.local.json`) では読み取られません。 * **自動続行されるのは事前選択中のオプション:** タイムアウト時に採用されるのは、その時点でダイアログ上に選択されているオプションです。意図しない選択肢で処理が進む可能性もあるため、長時間の自動作業や重要な判断が伴うタスクでは、タイムアウト値を長めに設定するか、`"never"` のままにしておくなど、タスクの内容に応じて使い分けることが推奨されます。 この `askUserQuestionTimeout` 設定は、Claude Codeを長時間の自律作業に利用する際に、質問で作業が止まり続けることを防ぐのに役立ち、エージェントに作業を任せて放置するワークフローとの相性が良いと言えます。ただし、質問への回答はユーザーの判断を反映する重要な要素であるため、自動続行に過度に依存せず、タスクの性質に応じてタイムアウト値を適切に設定することが重要です。
