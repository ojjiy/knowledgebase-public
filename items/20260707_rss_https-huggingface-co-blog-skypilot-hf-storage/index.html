Run AI workloads on any cloud, store on Hugging Face: zero-egress storage with SkyPilot

Date: 2026-07-07

Source: rss / Hugging Face Blog

Original URL

SkyPilotは、Hugging Faceストレージと統合され、AIワークロードを任意のクラウドで実行し、Hugging Faceにゼロエグラコストでデータを保存できる機能を提供します。 **主な機能と利点:** * **Hugging Faceデータとの統合:** * Hugging Faceのモデル、データセット、Spaceリポジトリ、またはBucketを、`hf://` URLと既存の`HF_TOKEN`を使用して、SkyPilotのジョブにマウント（読み書き可能または読み取り専用）できます。 * これにより、モデルやデータセットの保存場所をHugging Face Hubに保ちながら、SkyPilotで任意のクラウド上のGPUインスタンスで実行できます。 * **ゼロエグラコスト:** * Hugging Face Storageはエグラ料金を請求しないため、モデルやデータセットをGPUインスタンスに読み込む際のデータ転送コストが無料になります。これは、特に複数のクラウドにまたがってAIワークロードを実行する場合に、高額なエグラ料金を回避するのに役立ちます。 * AWS S3などと比較して、Hugging Face Storageはストレージ料金も安価です。 * **任意のクラウドでの実行:** * SkyPilotは20以上のクラウド、Kubernetes、Slurm、オンプレミス環境に対応しており、利用可能なGPUリソースがある場所でジョブを実行できます。 * これにより、特定のベンダーのインフラストラクチャに依存することなく、最適なGPUリソースを柔軟に利用できます。 * **Xetによる重複排除:** * Hugging Face BucketはXetテクノロジーを基盤としており、ファイルがコンテンツ定義チャンキングで分割され、重複するチャンクは一度だけ保存されます。 * これにより、差分のみが転送されるため、チェックポイント、モデルバリアント、データセットの追加などが効率化され、ストレージと転送コストが削減されます。 * **効率的なデータ読み込み（MOUNTモード）:** * MOUNTモードでは、`hf-mount` FUSEバックエンドが使用され、必要なデータチャンクのみがファイルシステムレイヤーで遅延して読み込まれます。 * これにより、GPUがすぐに作業を開始でき、データセット全体のダウンロードを待つ必要がなくなり、GPUのアイドル時間を削減できます。 * **簡単な認証:** * `HF_TOKEN`を環境変数に設定するか、`--secret HF_TOKEN`オプションで渡すだけで、認証が完了します。これにより、クラウドごとに異なるバケットキーを管理する必要がなくなります。 **利用開始方法:** 1. `pip install "skypilot[huggingface]"`でSkyPilotをインストールします。 2. `hf auth login`コマンドを実行するか、`HF_TOKEN`環境変数を設定して認証します。 3. SkyPilotのジョブ定義に`hf://`マウントを追加し、任意のエンドポイントで起動します。 この機能により、AIチームはデータとコンピューティングリソースの場所にとらわれず、コスト効率よく柔軟にAIワークロードを実行できるようになります。
