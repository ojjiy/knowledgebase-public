Copilot Studio の自律型エージェントの作成方法を整理してみた

Date: 2026-07-09

Source: rss / Qiita Trend

Original URL

Copilot Studio を用いて、ユーザーの操作なしに自律的に動作するエージェントを作成する方法について解説しています。 自律型エージェントとは、ユーザーがチャットを起動しなくても、メール受信やスケジュールなどのトリガーを基に自動的に情報調査や対応を行うエージェントのことです。 作成方法は、旧UIと新UIのワークフローで大きく分けて説明されています。 **旧UIでの作成方法:** * **トリガーの設定:** Power Automate を利用して、メール受信などのトリガーを設定します。 * **エージェントへの情報連携:** Power Automate のフローから「Sends a prompt to the specified copilot for processing」アクションで、トリガーで取得した情報をエージェントに渡します。 * **エージェントでの処理:** * **ツールの利用:** 回答メール送信などのタスクにはコネクタを利用します。 * **エージェントフロー:** 回答できない場合の問い合わせ起票や担当チームへの連絡などは、Copilot Studio 側で作成するエージェントフローで行います。 * **ライセンスとクレジット:** * トリガー用のフローは Power Automate のライセンス、エージェントフローは Copilot Studio のライセンスで動作します。 * エージェントフローの実行には、Microsoft 365 Copilot ライセンスを持つユーザーでも別途クレジットが消費される場合があります。環境に従量課金ポリシーを割り当てることで実行可能になるケースもあります。 * **公開の必要性:** 自律型エージェントを実行する前には、必ずエージェントを公開しておく必要があります。 **Power Automate のフローからエージェントを呼び出す方法:** * Power Automate のフローから特定のアクションを利用して、Copilot Studio のエージェントを直接呼び出すことも可能です。 * この方法で作成した場合、Power Automate のライセンスで動作します。 * エージェントの実行部分については、実行者が Microsoft 365 ライセンスを持っているかどうかで、環境のクレジット消費の有無が変わります。 * このアプローチは、費用感を把握しやすいと感じる人もいるかもしれません。 **新UIワークフローでの作成方法 (プレビュー):** * 新UIでは、エージェントフローは「ワークフロー」という名称に変わります。 * 作成画面が大きく変更され、Microsoft 365 Copilot Chat や Agent Builder で作成したエージェント、ファーストパーティのエージェントを呼び出すことが可能です。 * **シナリオ例:** Microsoft 365 Copilot のリサーチツールで調査し、その結果をHTMLに変換するエージェントを介してニュースレターを配信する、といったことが可能です。 * **自律型エージェントの位置づけ:** ワークフローでエージェントを呼び出す処理は、全て自律型エージェントとみなされます。 * **ライセンスとクレジット:** * ワークフローは Power Automate からは開けなくなり、Copilot Studio のライセンスで動作すると考えられています。 * アクションの実行には、別途クレジットを消費するという認識です。 **まとめ:** Copilot Studio で自律型エージェントを作成する方法は複数存在し、作成方法、利用するアクション、ライセンス、クレジット消費など、それぞれに違いがあります。今後は新UIのワークフローに集約されていく見込みですが、現時点では複数のアプローチがあるため、混乱を招く可能性があるため、本記事で整理が試みられています。
