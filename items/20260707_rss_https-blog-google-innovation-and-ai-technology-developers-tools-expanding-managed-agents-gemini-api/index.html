Expanding Managed Agents in Gemini API: background tasks, remote MCP and more

Date: 2026-07-07

Source: rss / Google AI Blog

Original URL

Gemini APIのマネージドエージェントが、バックグラウンド実行、リモートMCPサーバー連携、カスタム関数呼び出し、認証情報更新といった新機能によって拡張されました。これにより、開発者はより信頼性が高く、本番環境で利用できるエージェントを構築できるようになります。 **主な新機能:** * **バックグラウンド実行:** 非同期処理をサーバー側で実行できるようになり、HTTP接続を長時間維持する必要がなくなりました。APIは即座にIDを返し、クライアントはステータス確認や進捗ストリーミング、再接続が可能です。 * **リモートMCPサーバー連携:** プライベートデータベースや内部APIへのアクセスにカスタムプロキシミドルウェアを記述する必要がなくなり、マネージドエージェントを直接リモートのModel Context Protocol (MCP) サーバーに接続できるようになりました。Google検索やコード実行などの組み込み機能と組み合わせることで、エージェントは安全なサンドボックス環境からエンドポイントと通信できます。 * **カスタム関数呼び出し:** 組み込みサンドボックスツールと並行してカスタムツールを追加できるようになりました。APIはステップマッチングを使用し、組み込みツールはサーバーで自動実行されます。カスタム関数は、クライアントがローカルビジネスロジックを実行できるように、`requires_action`状態に遷移します。 * **ネットワーク認証情報更新:** アクセストークンやAPIキーの有効期限切れに対応し、既存の`environment_id`と新しいネットワーク構成を渡すことで、認証情報を更新またはローテーションできるようになりました。これにより、ファイルシステムの状態、インストール済みパッケージ、クローンされたリポジトリなどは維持されたまま、新しいルールが即座に適用されます。 これらのアップデートにより、マネージドエージェントはアプリケーションをブロックすることなく、実際の開発環境内で動作する非同期ワーカーへと進化しました。Gemini Interactions APIの概要やマネージドエージェントのクイックスタートガイドで、カスタムエージェント定義、環境構成、ネットワークルール、高度なストリーミングパターンなどを確認できます。
