SageMaker Endpoint に YOLO26 をホストしてみる

Date: 2026-06-26

Source: rss / DevelopersIO

Original URL

本記事では、SageMaker Endpoint に YOLOv26 の物体検出モデルをホストし、画像を入力として推論結果を取得するまでの手順が解説されています。 実施内容は以下の通りです。 1. **推論コンテナの作成**: * Dockerfile を使用して、Python 3.11-slim をベースに YOLOv26 の推論に必要なライブラリと `serve.py` を含んだコンテナイメージを構築します。 * モデルファイル (`best.pt`) はコンテナイメージには含めず、S3 上に配置し、SageMaker がエンドポイント起動時に `/opt/ml/model` ディレクトリに展開するように設定します。 * 依存関係は `requirements.txt` にまとめ、CPU 版 PyTorch を使用します。 2. **SageMaker 互換のエンドポイント実装 (`serve.py`)**: * SageMaker 推論コンテナで必須となるヘルスチェック用 `/ping` エンドポイントと、推論処理を行う `/invocations` エンドポイントを実装します。 * `/ping` ではモデルの読み込み可否を確認し、`/invocations` ではPOSTされた画像データをデコードし、YOLOv26 モデルによる物体検出を実行して結果を返します。 * 推論結果は、検出されたオブジェクトのラベル、クラスID、信頼度、バウンディングボックス情報を含む JSON 形式、または検出結果を描画した JPEG 画像として返却できます。 3. **モデルアーティファクトの作成**: * 学習済みのモデルファイル (`best.pt`) を `model.tar.gz` に圧縮し、S3 バケットにアップロードします。 4. **コンテナイメージのビルドと ECR へのプッシュ**: * 作成した Dockerfile からコンテナイメージをビルドし、Amazon ECR (Elastic Container Registry) にプッシュします。macOS でビルドする場合は `linux/amd64` プラットフォームを指定します。 5. **SageMaker Serverless Endpoint の作成**: * ECR にプッシュしたカスタムコンテナイメージと、S3 にアップロードしたモデルアーティファクトの URI を指定して SageMaker Model を作成します。 * エンドポイントは SageMaker Serverless Inference としてデプロイします。 6. **エンドポイントの呼び出し**: * SageMaker Runtime クライアントを使用して、作成したエンドポイントに画像を送信し、推論結果を取得します。 * `Content-Type` を `image/jpeg` とし、`Accept` ヘッダーで `image/jpeg` を指定すると、検出結果が描画された JPEG 画像を取得できます。JSON 形式で受け取る場合は、検出結果の詳細情報が含まれます。 最後に、SageMaker Endpoint と AWS Lambda の使い分けについても触れられており、軽量なモデルで推論時間が短い場合は Lambda、機械学習エンジニアとインフラエンジニアの役割分担が明確な組織では SageMaker Endpoint が適しているという私見が述べられています。
