Separating signal from noise in coding evaluations

Date: 2026-07-08

Source: rss / OpenAI News

Original URL

本記事は、コーディング評価における「SWE-Bench Pro」のタスクの品質について詳細な監査を行った結果を報告している。SWE-Bench Proは、より現実的で長期的なコーディングタスクを通じてモデルの能力を測定するために設計されたベンチマークだが、監査の結果、約30%のタスクに問題があることが判明した。 問題点は主に以下の4つのカテゴリに分類される。 1. **テストが厳しすぎる:** プロンプトで指定されていない実装詳細をテストが要求し、機能的に正しい提出が無効になる。 2. **プロンプトの要件が不明確:** テストによって実施され、推測が困難な要件がプロンプトから省略されている。 3. **テストのカバレッジが低い:** 要求された機能のチェックが不十分で、不完全な修正でも合格してしまう。 4. **プロンプトが誤解を招く:** モデルを誤った動作に誘導したり、テストの要求と矛盾する。 これらの問題は、厳密なコーディングベンチマークの作成の難しさを示唆しており、一方で、データ品質チェックのためのエージェントの有用性が高まっていることを示している。 本調査では、データ分析パイプラインと人間による検証を組み合わせた方法論を採用した。自動化された初期フィルタリングで問題のあるタスクを特定し、その後、エージェントによる詳細な監査と経験豊富なソフトウェアエンジニアによる人間によるアノテーションキャンペーンを実施した。人間によるレビューでは、エージェントよりも多くのタスクが「壊れている」と判断され、複数の問題が複合しているケースも多く見られた。 これらの結果を受けて、著者はSWE-Bench Proを推奨しないことを表明し、モデル開発者に対して慎重な結果の検証を求めている。評価は、不正行為が難しく、信頼でき、モデルの能力や整合性を真に反映するベンチマークを通じて、有意義なシグナルを提供する必要があると強調している。
