🤗 Kernels: Major Updates

Date: 2026-07-06

Source: rss / Hugging Face Blog

Original URL

🤗 Kernels プロジェクトは、カスタムカーネルのパッケージ化、配布、利用方法の標準化を目指しており、この度、セキュリティの向上、CLI の刷新、フレームワークやバックエンドの対応拡大、エージェント型カーネル開発の基盤構築など、大幅なアップデートが行われました。 主な変更点は以下の通りです。 **1. 新しいリポジトリタイプ「kernel」の導入:** Hugging Face Hub に「kernel」という新しいリポジトリタイプが追加され、アクセラレーター、オペレーティングシステム、バックエンドバージョンなどのカーネルの互換性情報が明示されるようになりました。これにより、カーネルの発見性が向上し、AI エコシステム全体に貢献します。 **2. セキュリティの強化:** * **信頼できるカーネル発行元 (Trusted Kernel Publishers):** デフォルトでは、信頼できる発行元からのカーネルのみがロードされるようになり、悪意のあるカーネルによるリスクが低減されました。信頼できる発行元になるには、コミュニティからの信頼を必要とし、申請制となります。 * **コード署名 (Kernel Signing):** カーネル開発者のみが持つ秘密鍵でカーネルに署名し、公開鍵で検証する仕組みが導入されました。これにより、Hub の認証情報が漏洩した場合でも、悪意のあるカーネルの署名を偽造することが困難になります。Sigstore の cosign を利用し、一時的な秘密鍵を使用することで、さらにセキュリティを高めています。 **3. CLI の刷新:** `kernels` と `kernel-builder` の CLI が再設計され、関心の分離が明確になりました。`kernels` はカーネルのロードと準備に特化し、`kernel-builder` はカーネルのビルドに特化することで、それぞれがより軽量かつ具体的になりました。 **4. フレームワークとバックエンドの対応拡大:** * **Torch Stable ABI のサポート:** カーネル開発者が特定の Torch バージョン以降をターゲットにできるようになりました。 * **Apache TVM FFI のサポート:** PyTorch, Jax, CuPy などのフレームワーク間で相互運用可能なカーネルを開発できるようになりました。 **5. エージェント型カーネル開発の基盤構築:** `kernel-builder` と `kernels` は、エージェントがゼロから最適化されたカーネルを開発するワークフローをサポートします。エージェントは、カーネルの雛形作成、ビルド、ベンチマーク、反復的な最適化を行うことができ、Hugging Face Jobs との連携により、様々なハードウェア構成でのベンチマークが容易になり、パフォーマンスのフィードバックを得やすくなりました。 **その他の改善点:** * **環境構築スクリプト:** カーネルビルド環境のセットアップを簡略化するインストールスクリプトが提供されました。 * **システムカード:** ビルドされたカーネルに関する有用な情報（使い方、インターフェースなど）を公開するシステムカードが生成されます。 * **互換性チェック機能:** `has_kernel()` メソッドでカーネルの互換性を確認したり、`get_kernel_variants()` メソッドで互換性の有無とその理由を確認できるようになりました。 * **manylinux_2_28 のサポート改善:** libstdc++ の動的リンクにより、互換性の問題が解消され、より安定したビルドが可能になりました。 これらのアップデートにより、🤗 Kernels プロジェクトは、カーネル開発者とカスタムカーネルのユーザー双方にとって、より使いやすく、安全で、強力なプラットフォームへと進化しました。
