シニアエンジニアがコードをほぼ書かなくなった理由

Date: 2026-06-26

Source: rss / Zenn LLM

Original URL

ソフトウェアエンジニアのキャリア15年を持つ筆者は、AIの進化、特にAnthropicのClaude Code 2.0のVSCode拡張機能の本格化（2025年9月）をきっかけに、自身の業務内容が大きく変化したと述べています。以前はコードを書くことが中心でしたが、現在はAIがコードの大部分を生成し、自身は仕様書の作成、AIが生成したコードのレビュー、アーキテクチャやテスト設計における最終的な意思決定といった「上流工程」に集中しています。 この変化により、残業時間は大幅に削減され、生産性は飛躍的に向上しました。筆者は、AIがコードを書く一方で、人間は「仕様を書く」「AIが書いたコードを読む」という役割分担が明確になったと指摘しています。特に、AIが生成したコードを鵜呑みにせず、その品質や意図を正確に理解し、最終的な責任を持つ「コードを読む力」が最も重要なスキルとなったと強調しています。 AIの利用において、仕様書の曖昧さがコードの品質に直結すること、AIがエラーを隠蔽してしまうといった課題も浮き彫りになりました。これらの課題に対しては、AIにエラーハンドリングの記録や学習を促す仕組み（lessons.md）を導入したり、AIによるテスト実装の前に人間がテスト設計を行うといった対策が講じられています。 AI駆動開発は、個人の生産性向上に留まらず、チーム全体への波及も期待されています。筆者は、AIとの協働で「考える時間」が増え、T型人材としての幅広い知識と専門性を活かしてAIの成果物を判断する能力が、これからのエンジニアに求められると結論付けています。AIはあくまで手段であり、何を作るべきかを決めるのは人間であり、AIを使いこなすことで日本のIT人材不足の解決にも繋がると見解を示しています。
