Bedrock Mantle で Claude Fable 5 を呼び出せるようになっていたので試してみた

Date: 2026-07-05

Source: rss / DevelopersIO

Original URL

Amazon Bedrock Mantle で Claude Fable 5 モデルが利用可能になったため、その検証結果について報告します。 7月1日時点では、Mantle で Claude Fable 5 モデルは存在せず、404エラーが返されていました。しかし、7月5日にはモデル情報が取得できるようになり、Data Retention Mode を設定することで呼び出しに成功しました。 **変更点:** * **Mantleでのモデル存在:** 7月1日時点では404エラーでしたが、7月5日には `GET /v1/models/{id}` でモデル情報が取得可能になりました。 * **`/anthropic/v1/messages` への試行:** Data Retention 設定後に成功しました。 * **OpenAI互換パス:** モデルが存在しないため試行できませんでしたが、今回確認したところ非対応（400エラー）であることが判明しました。 **検証内容:** 1. **モデルの公開状況確認:** `GET /v1/models/anthropic.claude-fable-5` でモデル情報を確認したところ、`status` が `unavailable` で、`data_retention.allowed_modes` に `["provider_data_share"]` のみが設定されていました。`inherit` のままでは利用できない状態でした。 2. **Data Retention Mode の設定:** * Mantle 側の現在の設定を確認したところ `inherit` でした。 * `inherit` のままでは Fable 5 の呼び出しに失敗するため、Mantle 側の Data Retention Mode を `provider_data_share` に変更しました。 * 参考として、Control Plane 側の Data Retention は 6月10日に `provider_data_share` に設定済みでしたが、Mantle 側は別管理でした。 3. **Anthropic Messages API での呼び出し:** Data Retention 設定後、`/anthropic/v1/messages` エンドポイントから Claude Fable 5 を呼び出すことに成功しました。 4. **Anthropic Python SDK での呼び出し:** Anthropic SDK を使用し、`base_url` に Mantle の `/anthropic` までを指定することで、同様に呼び出しが可能でした。 5. **OpenAI 互換パスの非対応:** Claude Fable 5 は Mantle の OpenAI 互換エンドポイント (`/openai/v1/responses` や `/v1/chat/completions` など) には対応しておらず、いずれも HTTP 400 エラーが返されました。 **まとめ:** 7月1日時点では Mantle で利用できなかった Claude Fable 5 が、7月5日には利用可能になっていました。Mantle 側の Data Retention Mode を `provider_data_share` に変更することで、`/anthropic/v1/messages` エンドポイントから呼び出すことができました。Control Plane 側の Data Retention 設定とは別管理であるため、Converse API で利用できている場合でも Mantle 側の設定確認が必要です。
