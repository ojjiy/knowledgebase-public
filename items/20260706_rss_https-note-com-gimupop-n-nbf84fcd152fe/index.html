「レビューするくらいなら自分でやった方が早い」問題について｜柳川慶太

Date: 2026-07-06

Source: rss / Hatena Bookmark Programming

Original URL

「レビューするくらいなら自分でやった方が早い」という問題は、AIの進化により、その構造が不可逆的に変化しようとしています。 従来のレビューには、 1. **指摘場所を探すコスト**: 作成側は自身の作業範囲のみを見れば良いのに対し、レビュー側は通らなかった道も含めて穴を探す必要があり、多大な集中力を要します。 2. **翻訳するコスト**: 頭の中の「ここが違う」という認識を、相手に伝わる言葉（理由、改善策、自身の考え）に変換する作業に多くの時間が費やされます。単に正しく伝えるだけでなく、相手の感情に配慮したトーン調整も必要です。 3. **嫌われるコスト**: レビュアーは、たとえ相手のためであっても、指摘を受ける側から気分を害されることが多く、関係性に影響を与える可能性があります。 これらのコストを乗り越えてレビューを行う目的は、 1. **品質の担保**: 成果物の水準を守り、多様な視点を取り入れてより良いものにするため。 2. **育成**: 指摘を通じて、相手が将来的に自分で気づけるようになることを期待するため。 3. **責任の所在**: 「これで出して良い」と、誰かが最終的な決定と責任を引き受けるため。 しかし、これらの役割分担がうまく機能するには、「スキルが拮抗している」「お互いの視点が補い合える」「時間軸が噛み合っている」という条件が必要であり、これらが揃わない場合は「自分でやった方が早い」という状況に陥りがちです。 AIの登場は、このROI（投資対効果）の計算を大きく変えました。AIを使えば、個人で作成できる範囲が広がり、レビューで指摘する作業とAIへの指示が類似していることに気づき始めます。その結果、「人に任せてレビューするくらいなら、AIで自分で作った方が早い」という計算に傾きやすくなります。レビュイーが指摘事項をそのままAIに渡して修正させるような行為は、レビューの往復による効率を低下させ、レビュアーのモチベーションを削ぐ要因となります。 そして、この記事で最も伝えたいのは、「レビューできる仕事は基本的にAIに代替されていく」という事実です。なぜなら、「レビューできる」ということは、何が正しく何がダメかの「基準」があることを意味します。基準があるならば、AIに作成させ、その結果を自分でチェックする方が、人間に依頼して往復するよりも速く、安く、漏れなく実現できる場面が増えるからです。 ただし、これは「ゼロイチ」の話ではなく、「グラデーション」が存在します。 * **AIが強い領域**: 論点が出そろっており、具体的に指示を切り出せる仕事。 * **人が強い領域**: 論点が曖昧なまま渡されても、何を確認すべきかを自分で考え、論点を立てて進められる仕事。 しかし、この境界線はAIの進化とともに常に押し上げられており、「今は勝てている」という状況に安心せず、より抽象的なレベルで仕事を進める能力を磨き続ける必要があります。 レビューの目的をAIに当てはめると、 * **品質の担保（作業）**: AIが代替し、人よりも速く正確になる可能性が高い。 * **育成（一部代替）**: AIが丁寧にフィードバックを返せるなら、それで学ぶことも可能。 * **責任の所在**: 「これで出していい」と引き受けることは、AIにはできない領域であり、人に残ります。 AIが代替するのは「作業」であり、AIが代替しないのは「引き受け」です。人が責任を引き受けられるのは、事業の狙い、過去の経験、メンバーの状況、将来への影響といった「長い文脈」を持っているからです。文脈を持たない人の「OK」は単なるハンコであり、AIで代替可能になる可能性があります。 AIと協働する時代においては、あなたがAIに指示を出し、その結果に対して「どこが目的とずれているかを**探す**」「直してほしいことを伝わる言葉に**翻訳する**」「何度も往復して**形にする**」という作業は、もはやあなた自身が行う「レビュー」そのものです。つまり、作る人とレビュアーの役割が一人の中に同居する時代になったのです。 ただし、誤字脱字の指摘や表記ゆれの統一といった「表面的なチェック」は、AIの方が速く正確であるため、人が行う意味が薄れていきます。人が残るレビューは、「何のために」「どこが詰まっているのか」を問い直し、基準そのものを持ち込むような、より深いレベルのものです。 AI相手であっても、最も効果的なのは「ナラティブを語れるか」です。つまり、「何のために」「何を」「どこがボトルネックか」を理解し、それを言葉にして形にする力です。これは「文脈を持つ」ことと同義であり、責任を引き受けるための基盤となります。 レビューは、一方的な指摘ではなく、文脈を渡し合う「助言の往復」へと変化します。AIが表面的なチェックを担うことで、人は本来のレビューの姿、すなわち「目的の共有」や「ボトルネックの把握」といった、より本質的な部分に立ち戻ることができるのです。 最終的に、AI時代に私たちが育てるべきは「作業の速さ」ではなく、「長い文脈を抱え続ける力」「ナラティブを語る力」「それを足場に引き受ける覚悟」です。これらは、レビュアーだけでなく、指示を受ける側も同様に重要となります。 [番外編] AIの活用によりアウトプットが長大化する問題に対しては、レビュアーとレビュイー双方が「どこまで見て、どこまで見ないか」のコンセンサスを揃える工夫が求められます。コードレビューの文化のように、レビュー範囲の宣言や、見てほしい箇所の明確な伝達が重要となります。 「レビューするくらいなら自分でやった方が早い」という問題は、AIの登場により構造的に変化しました。この変化を直視し、文脈を渡し合う「レビュー」の本質を理解し、AI時代に求められる能力を磨いていくことが、今後のキャリアにおいて不可欠となるでしょう。
