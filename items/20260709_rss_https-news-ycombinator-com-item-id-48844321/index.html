LLM-as-a-Verifier: A General-Purpose Verification Framework

Date: 2026-07-09

Source: rss / Hacker News LLM

Original URL

本論文は、「LLM-as-a-Verifier」という、汎用的な検証フレームワークを提案する。大規模言語モデル（LLM）の能力向上には、事前学習、事後学習、テスト時計算能力の拡張が中心的なパラダイムとなっているが、本研究では「検証」、すなわち解の正しさを判断する能力を新たなスケーリング軸として特定した。 「LLM-as-a-Verifier」は、追加学習を必要とせずに、エージェントタスクに対してきめ細やかなフィードバックを提供する。従来のLMジャッジが候補解に対して離散的なスコアを生成するようLLMに指示するのとは異なり、本フレームワークはスコアリングトークン対数（logits）の分布に対する期待値を計算することで、連続的なスコアを生成する。この確率的定式化により、検証は以下の複数の次元でスケーリング可能となる。(1)スコアの粒度、(2)繰り返し評価、(3)基準分解。 特に、スコアリング粒度をスケーリングすることで、肯定的な解と否定的な解との間の分離が向上し、よりキャリブレーションされた比較が可能になることが示されている。さらに、繰り返し評価と基準分解のスケーリングは、分散と複雑性を低減することで、検証精度を継続的に向上させる。また、検証器の連続スコアを用いた候補解の中から最良の解を選択するための、コスト効率の高いランキングアルゴリズムも導入されている。 「LLM-as-a-Verifier」は、Terminal-Bench V2（86.5%）、SWE-Bench Verified（78.2%）、RoboRewardBench（87.4%）、MedAgentBench（73.3%）において、最先端の性能を達成した。検証機能以外にも、「LLM-as-a-Verifier」からのきめ細やかな信号は、タスクの進捗を推定するための代理としても機能する。開発者が自身のエージェントシステムを監視・改善できるように、Claude Codeの拡張機能が構築された。さらに、「LLM-as-a-Verifier」は強化学習（RL）に対して密なフィードバックを提供でき、ロボティクスおよび数学的推論ベンチマークにおけるSACおよびGRPOのサンプル効率を改善できることが示されている。
