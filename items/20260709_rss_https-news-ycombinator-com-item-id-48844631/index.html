Show HN: A control-theory approach to detecting LLM agent instability

Date: 2026-07-09

Source: rss / Hacker News LLM

Original URL

「state-harness」は、LLM（大規模言語モデル）エージェントのマルチターンの対話における不安定性を検出し、その原因を特定するための制御理論に基づいたツールです。具体的には、トークン使用量の急増（「トークン・スパイラル」）を監視し、失敗パターン（コンテキスト蓄積スパイラル、リトライ・ストーム、ポリシー・ドリフトなど）を分類します。 このツールは、Lyapunov安定性監視の概念を応用し、エージェントのトークン消費量をベースラインと比較して「エネルギー」としてモデル化します。エネルギーの増加率が一定期間閾値を超えると、不安定と判断して監視をトリップさせます。トリップした際には、追加のLLM呼び出しなしに、失敗の原因、発生したパターン、それに伴うコスト増加の見積もり、そして具体的な改善策（例：履歴圧縮、閾値調整、スライディングウィンドウ戦略の導入）を提示します。 「state-harness」は、特にMCTS（モンテカルロ木探索）やビームサーチなどの探索ツリー型エージェントや、大規模なプラットフォームチームでのエージェントの異常検知、ベンチマークの信頼性向上に役立ちます。 **主な機能と特徴:** * **不安定性の検出:** トークン使用量の成長率を監視し、異常な増加を検出します。 * **失敗パターンの分類:** コンテキスト・スパイラル、リトライ・ストーム、ポリシー・ドリフトなど、一般的な失敗パターンを特定します。 * **原因究明:** 追加のLLM呼び出しなしに、失敗の原因を分析し、人間が理解しやすいレポートを生成します。 * **改善策の提案:** 特定された失敗パターンに基づいて、具体的な改善策を提案します。 * **低オーバーヘッド:** 既存のエージェントループに容易に統合でき、LLM呼び出しの追加がないため、計算コストを抑えられます。 * **フレームワーク統合:** LangGraph、CrewAIなど、主要なLLMエージェントフレームワークとの連携が可能です。 * **ベンチマークによる検証:** 複数のベンチマークで、その有効性と非侵襲性が検証されています。 「state-harness」は、マルチターンLLMエージェントが抱える、原因不明の失敗やコストの増大といった課題に対処するための強力なソリューションを提供します。
