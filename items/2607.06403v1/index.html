# From Foundation to Application: Improving VLA Models in Practice [View Paper](http://arxiv.org/abs/2607.06403v1) ## 1. 問題意識：解決したい問題はなにか 近年、Vision-Language-Action (VLA) モデルは、ロボット制御における汎用的な基盤モデルとしての可能性を示していますが、実験室環境での性能と実際の現実世界での応用との間には大きな乖離があります。この乖離は、ロボットがより多様なロボット構成、より広いアクション空間、そしてより動的な環境で動作する必要があるという現実世界の要求を満たせないことに起因します。具体的には、既存のVLAモデルは、以下のような現実世界の課題に対応できていません。 * **異種ロボット構成への対応不足**: 実世界では、様々な形状や機能を持つロボットが存在しますが、既存モデルはこれらの多様なロボット構成間での汎化が十分ではありません。 * **限定的なアクション空間**: 多くの現実世界のロボットは、単なるアーム操作だけでなく、頭部、腰、移動台、器用な手など、より多くの自由度（Degrees of Freedom; DoF）を持っています。既存モデルはこの拡張されたアクション空間を十分に活用できません。 * **動的な環境における時間的推論の弱さ**: 現実世界では、環境の変化を予測し、将来の状況を考慮した行動計画が不可欠です。しかし、既存モデルは現在の観測にのみ反応する傾向があり、将来のシーン進化や行動結果を予測する能力が限定的です。 これらの課題は、現在のVLA基盤モデルの実用性を著しく制限しており、現実世界でのロボット応用を妨げています。 ## 2. 提案手法のアプローチ：どのようなアイデア・アプローチで問題を解決しようとしたか 本研究では、これらの課題を解決するために、LingBot-VLA 2.0 という VLA モデルを提案し、特に以下の3つの機能領域で改良を加えています。 1. **タスクおよびロボット構成間での汎化性能の向上**: * データ処理パイプラインを刷新し、大規模な事前学習コーパスを構築しました。このコーパスには、約60,000時間のデータが含まれています。 * 具体的には、20種類のロボット構成にわたる約50,000時間のロボット軌跡データと、10,000時間の自己視点（egocentric）人間ビデオデータを使用しました。これにより、ロボットの多様な動きや人間とのインタラクションシナリオを網羅し、汎化性能を高めています。 2. **アクション空間の拡張（デュアルアームハードウェアプラットフォームを超えて）**: * 標準的なデュアルアームロボットプラットフォームに加え、頭部、腰、移動台、器用な手といった、より多くの自由度（DoF）を制御できるようにシステムを拡張しました。 * これにより、ロボットはより複雑なタスクに対応できるようになり、現実世界の多様なシナリオでの応用が可能になります。 3. **予測的ダイナミクスモデリングによる時間的推論の改善**: * 将来の予測をプロキシタスクとして定式化しました。 * この予測を支援するために、意味論的な事前情報を提供するビデオ表現モデルと、幾何学的な手がかりを提供する深度推定モデルを活用しました。 * これにより、VLAモデルは将来のシーン変化や行動結果をより良く推論できるようになり、動的な環境での意思決定能力が向上します。 これらの改良は、VLA基盤モデルの能力を、現実世界でのロボット応用に向けて、より実用的なものへと橋渡しすることを目的としています。 ## 3. 成果：提案手法により、何が達成できたのか 提案手法である LingBot-VLA 2.0 は、以下の成果を達成しました。 * **GM-100 ベンチマークでの性能向上**: 一般的な設定で実施された GM-100 ベンチマーク評価において、提案された3つの改良（汎化、アクション空間、予測的ダイナミクスモデリング）が、実用的な能力を確実に向上させることが検証されました。 * **強力なクロスエンボディメント（異種ロボット間）長時限モバイル操作能力**: 全身の自由度を網羅した拡張された事前学習データを利用することで、LingBot-VLA-2.0 は2つの異なるロボットプラットフォーム間で、強力なクロスエンボディメント長時限モバイル操作能力を示しました。 * **bimanual（両手）操作における優れた性能**: 一般的な設定での両手操作タスクにおいて、LingBot-VLA 2.0 は、両方のロボットプラットフォームで最高の総合性能を達成しました。具体的には、Agilex Cobot Magic では LingBot-VLA 1.0 を 8.0/4.4 ポイント（進行スコア/成功率）、[先行研究] を 7.1/2.2 ポイント上回りました。Galaxea R1 Pro では、[先行研究] を 7.2/6.7 ポイント上回りました。 * **長時限モバイル操作における汎化能力**: 2つのロボットプラットフォームにおける長時限モバイル操作タスクにおいて、LingBot-VLA 2.0 は、ドメイン内（ID）およびドメイン外（OOD）の両方の設定で、[先行研究] を一貫して上回りました。特にOOD設定では、成功率において LingBot-VLA 2.0 がより堅牢であり、タスク完了能力が高いことが示されました。 * **よりオブジェクト中心で目標志向の操作**: 特にオブジェクトの正確な位置特定と目標指向の行動実行が求められるタスクにおいて、LingBot-VLA 2.0 は大幅な改善を示しました。これは、より強力な VLM バックボーン、強化された位置特定能力、および将来の情報に基づいた行動予測によるものです。 これらの結果は、LingBot-VLA 2.0 が、VLA基盤モデルを実験室での成功から現実世界での応用へと進めるための、実用的な方向性を示していることを強調しています。 ## 4. 課題：本手法の限界・課題にはなにか。本文中で著者が言及しているものと、あなたが想定するもの ### 著者が言及している課題 * **タスクごとの性能のばらつき**: 全てのタスクで一様に改善が見られるわけではなく、特に最終的な正確な配置、解放、または完了ステップで失敗するタスクが存在し、進行スコアと成功率の間に大きなギャップが見られました。 * **エンボディメント特有の要因**: Agilex Cobot Magic と Galaxea R1 Pro の間で性能の格差が見られました。これは、運動学、カメラ視点、アクション空間の整合性といった、ロボット固有の要因が依然として課題であることを示唆しています。 * **OOD設定での性能低下**: 長時限モバイル操作タスクにおいて、OOD設定（ロボットの初期位置の摂動、未知のオブジェクトカテゴリ）では、ID設定と比較して性能が低下しました。これは、未知の状況への汎化能力にまだ改善の余地があることを示しています。 * **Refrigerator Sorting タスクにおけるOODでの大きな性能低下**: Refrigerator Sorting タスクでは、IDからOODへの性能低下が Stove Cleaning タスクよりも大きかったと報告されています。これは、OOD設定が初期ロボット姿勢と操作対象オブジェクトカテゴリの両方を同時に変更するため、より高度なオブジェクトレベルの汎化と長時限でのリカバリ能力が必要とされるためです。 ### 想定される課題 * **計算コスト**: 大規模なデータセットでの事前学習や、複雑なモデル構造（例: 予測的ダイナミクスモデリング）は、依然として高い計算リソースを必要とする可能性があります。 * **データ収集とアノテーションのコスト**: 60,000時間という膨大な事前学習データ（特にロボット軌跡データ）の収集と品質管理には、多大な時間と労力がかかることが予想されます。 * **未知のタスクや環境への対応**: 現在の改善は、既存のベンチマークや特定の現実世界シナリオに焦点を当てていますが、完全に未知のタスクや、予測が困難な極端な環境変化に対して、どの程度頑健に機能するかは不明です。 * **安全性と信頼性**: 現実世界でのロボット応用においては、安全性と信頼性が最重要です。提案手法が、予期せぬ動作や危険な行動を回避できるかどうかの検証は、さらなる評価が必要です。 * **人間とのインタラクションの複雑さ**: 自己視点人間ビデオデータを使用していますが、人間とのより微妙で複雑なインタラクション（例: コミュニケーション、意図の理解）への対応は、さらなる研究が必要かもしれません。 ## 5. 技術詳細：提案手法の技術的な詳細について、できるだけ詳しく。数式やアルゴリズム、モデル構造なども可能な限り詳しく LingBot-VLA 2.0 は、以下の主要な技術的要素によって構成されています。 ### 1. 大規模事前学習データセットの構築 * **データソース**: * **ロボット軌跡データ**: 20種類の異なるロボット構成（単腕、両腕、移動台付きロボットなど）から収集された約50,000時間の高精度なロボット軌跡データ。 * **自己視点人間ビデオデータ**: 10,000時間の自己視点（egocentric）人間ビデオデータ。これらは、人間が体験する視点からの操作を捉えています。 * **データ前処理パイプライン**: * **軌跡スムーズネス評価**: 軌跡のジャーク（3次微分の変化）、速度、加速度のZスコアを計算し、閾値を超える軌跡を除外します。 * **静止期間の検出**: 状態と行動信号がほとんど変化しない期間がエピソードの95%以上を占める場合、そのエピソードを除外します。 * **ビデオと状態信号の整合性検証**: URDF（Unified Robot Description Format）を用いてロボットを画像平面に投影し、記録された状態とビデオとの一致を確認します。不整合がある場合は除去します。 * **ビデオ品質フィルタリング**: 人手によるアノテーションプロセスで、ぼやけ、重度のオクルージョン、フレーム落ち、複数視点の不整合があるビデオを除外します。 * **自己視点人間ビデオの処理**: * **VLM事前フィルタリング**: ビデオレベルのVLM（Vision-Language Model）を使用して、自己視点操作の前提を満たさないビデオ（例: 第三者視点、歩行動画、明確な手と物体のインタラクションがない、操作対象物がない）を除外します。 * **SLAMと手ポーズ推定**: 自己視点SLAM（Simultaneous Localization and Mapping）によりカメラの内部・外部パラメータを推定し、手ポーズ推定によりMANOパラメータを復元します。 * **標準化された手軌跡フォーマット**: 既存のアクション/手軌跡ラベル付きデータと、推定された手軌跡データを、世界座標系における時間的に連続した手軌跡として統一します。 * **軌跡レベルの品質管理**: 十分な有効フレーム率（20%以上）、安定したSLAM軌跡、突然の不連続性がない手軌跡、人間の生理学的制約を violation しないサンプルのみを採用します。 ### 2. 拡張されたアクション空間 * **制御自由度**: 頭部、腰、移動台、器用な手など、より多くの自由度（DoF）の制御をサポートします。 * **統一された状態・行動表現**: * 状態と行動は、55次元の標準化されたベクトル表現で表されます。 * 内訳: アーム関節位置 (14次元)、エンドエフェクタ姿勢 (14次元)、グリッパー位置 (2次元)、手関節位置 (12次元)、腰位置 (4次元)、頭部位置 (2次元)、移動信号 (3次元)。残りの4次元は予約されています。 * **アーム関節とエンドエフェクタ姿勢**: デュアルアーム構成の最大次元数に対応するように定義されています。エンドエフェクタ姿勢はXYZ座標と回転クォータニオン（7次元）で表現されます。単腕データの場合は、次元がパディングされます。 * **異種エンボディメントのマッピング**: 異種エンボディメントの制御は、共有ボディパーツコンポーネントで構成されるコンパクトな行動ベクトルにマッピングされます。 ### 3. 予測的ダイナミクスモデリング * **プロキシタスク**: 将来の予測をプロキシタスクとして定式化し、VLAモデルの時間的推論能力を向上させます。 * **教師モデル**: * **深度推定モデル**: シーンの幾何学的な情報を捉えるために使用されます。 * **ビデオ表現モデル (DINO-Video)**: 意味論的な事前情報と、因果的な時間的注意機構を備えたモーションアウェアな表現を提供します。DINOv3 をベースに、ブロックごとの因果的時系列注意機構と3Dロータリー位置埋め込み（3D-RoPE）を導入しています。 * **二重クエリ蒸留（Dual-Query Distillation）**: * `Q_t` (現在のクエリ) と `Q_{t+T}` (将来のクエリ) という2つの学習可能なクエリを、視覚的・テキスト的トークンに追加します。 * `Q_t` は現在の深度 (`D_t`) および DINO-Video 特徴 (`Z_t`) を予測するように学習されます。 * `Q_{t+T}` は将来の深度 (`D_{t+T}`) および DINO-Video 特徴 (`Z_{t+T}`) を予測するように学習されます。 * **損失関数**: ```python # Depth Distillation Loss loss_depth = mean( abs(projection_depth(Q_t) - D_t) + abs(projection_depth(Q_{t+T}) - D_{t+T}) ) # Video Distillation Loss loss_video = mean( squared_error(projection_video(Q_t) - Z_t) + squared_error(projection_video(Q_{t+T}) - Z_{t+T}) ) ``` ここで、`projection_depth` および `projection_video` は、次元整合のためのクロスアテンションを持つモジュールです。 ### 4. Mixture-of-Experts (MoE) アーキテクチャ * **目的**: VLAシステムのアクションモデリング容量をスケーリングし、異種エンボディメント間での汎化を促進するために使用されます。 * **トークンレベルの損失フリーMoE**: * アクションエキスパート内のフィードフォワードネットワーク（FFN）を、トークンレベルのスパースMoEレイヤーで置き換えます。 * **共有エキスパート**: 全てのトークンで共通の汎用的プライオリティを学習します。 * **ルーティングエキスパート**: 各トークンの特徴に基づいて、適応的に選択される専門化されたモデリング能力を提供します。 * **ルーティングメカニズム**: ソフトマックス正規化による強い競争を避けるため、シグモイド活性化関数とルーティング補正バイアス（`b_l,j`）を用いて、各トークンが独立して複数のエキスパートをアクティブ化できるようにします。 * **損失フリー負荷分散**: 負荷分散をプライマリアクション学習目的から分離し、ルーティング補正バイアスを調整することで、エキスパートの利用を均衡させます。 ```python # Expert Output Calculation (Simplified) # m_l(u_l,t) is the output of the l-th MoE layer for token u_l,t # E_l^(s) is the shared expert, E_l,j^(r) is the j-th routed expert # g_l,j is the mixture weight, R(u_l,t) is the set of selected experts m_l(u_l,t) = E_l^(s)(u_l,t) + lambda * sum(g_l,j(u_l,t) * E_l,j^(r)(u_l,t) for j in R(u_l,t)) # Load Balancing Bias Update # n_l,j is the count of tokens assigned to expert j # N_r is the total number of routed experts b_l,j_new = b_l,j_old - gamma * sign(n_l,j - mean(n_l,k for k in all experts)) ``` * **利点**: 限定された計算予算内で、より効率的なスケーリングと、より低い学習損失および検証誤差を達成できます。 ## 6. この論文を140字以内のツイートで要約すると？ 現実世界で動くロボットのVLAモデルを改良🤖。大規模データと拡張アクション空間、未来予測能力を強化した「LingBot-VLA 2.0」を開発。異種ロボット間での長距離操作や複雑なタスクで性能向上！ #ロボティクス #AI
