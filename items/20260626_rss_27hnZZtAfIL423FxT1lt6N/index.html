Prettierからoxfmtに乗り換えてimport sortの設定をする

Date: 2026-06-26

Source: rss / DevelopersIO

Original URL

この記事では、JavaScript/TypeScriptのコードフォーマッターであるPrettierからoxfmtへの乗り換えについて解説されています。 **乗り換えの背景:** * 筆者がViteでWebアプリ開発中に、create-viteのデフォルトリンターがoxlintになったことを知った。 * oxlintの開発元が提供するoxfmtがimport sortに対応しており、Prettierのプラグインでimport sortを行っていたため、乗り換えを検討した。 **oxfmtとは:** * Rustで書かれたJavaScript/TypeScript向けの高速ツール群「Oxc」の一部。 * Prettier互換の高速フォーマッターであり、Prettierの30倍、Biomeの3倍の速度で動作するとされている。 * 最大の利点は、import sortに組み込みで対応していること。 **導入手順:** 1. **環境構築:** ViteでReact+TypeScriptのプロジェクトを作成。 2. **oxfmtのインストール:** `npm install -D oxfmt` 3. **初期設定:** `npx oxfmt --init` を実行し、`.oxfmtrc.json` を生成。 4. **import sortの設定:** * デフォルトではimport sortは無効。 * `.oxfmtrc.json` に `sortImports` の項目を追加し、設定を行う。 * `sortImports.newlinesBetween: true` でグループ間に空行を挿入。 * `sortImports.customGroups` で `@/**` のようなカスタムグループを定義。 * `sortImports.groups` で import のソート順を設定 (builtin, external, alias, parent, sibling, index, unknown)。 **試用結果:** * 設定後、oxfmtを実行すると、import文のソートが意図した通りに行われた。 * Prettierと異なり、oxfmtはパスを指定しなくても対象ファイルを自動で探し、更新してくれる。 **まとめ:** * oxfmtはデフォルトでimport sortに対応しており、Prettierからの乗り換えが容易である。 * PrettierやBiomeの設定ファイルをインポートする `--migrate` コマンドも提供されている。 **おまけ:** * `npx oxfmt --migrate=<SOURCE>` コマンドでPrettierやBiomeの設定をインポートできる。ただし、筆者の場合はプラグイン設定のため、うまく機能しなかった。
