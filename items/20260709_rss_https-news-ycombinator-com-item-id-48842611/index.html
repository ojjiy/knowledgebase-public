Why developers are ditching GitHub for Codeberg and self-hosting alternatives

Date: 2026-07-09

Source: rss / Hacker News Front Page

Original URL

GitHubは依然として人気が高いものの、一部の開発者は頻繁なダウンタイムや、Microsoftによる買収後の政治的な方向性など、様々な懸念からGitHubからの移行を検討しています。Ghostty（ターミナルエミュレーター）、Zig（システムプログラミング言語）、Tenacity（オーディオエディター）といった著名なプロジェクトがGitHubを離れ、Codebergのような代替サービスや自己ホスティングを選んでいます。 開発者がGitHubを離れる主な理由は以下の通りです。 * **技術的な問題:** 頻繁なダウンタイムが最も一般的な不満であり、GhosttyやZigの移行の主な要因となっています。 * **政治的な理由:** GitHubの特定の組織との関係性などが、一部の開発者にとって懸念材料となっています。 * **AIの統合:** GitHub CopilotのようなAIツールの急速な統合と、それに対するCEOの強硬な姿勢が、一部の開発者から反発を受けています。 これらの理由に加え、GNUプロジェクトのように、そもそもGitHubが非フリーソフトウェアを必要とすることや「悪いライセンス慣行」を推奨することなどを理由に、GitHubを避けてきたプロジェクトも存在します。 GitHubの代替手段としては、Codeberg（非営利）、GitLab（自己ホスティングオプションあり）、Bitbucket、Sourcehut（完全オープンソース）、GiteaやForgejo（自己ホスティング）、さらにSoftware Freedom Conservancyのような支援団体による移行支援キャンペーンなど、多様な選択肢があります。 GitHubはオープンソース開発の促進に大きく貢献していますが、問題点も存在します。技術的な問題であれ、思想的な問題であれ、開発者には様々な代替手段が利用可能であることを認識することが重要です。
