# タイトル Hallucination in World Models is Predictable and Preventable [View Paper](http://arxiv.org/abs/2606.27326v1) ## 1. 問題意識：解決したい問題はなにか 現代の生成型ワールドモデルは、現実的で操作可能な未来の映像を生成する能力が向上していますが、しばしば「幻覚（hallucination）」を起こします。これは、生成される映像が視覚的には流暢で自然に見えるものの、現実世界の真のダイナミクスから逸脱してしまう現象です。この幻覚は、生成された誤った未来情報が下流のプランニングやポリシーに直接影響を与えるため、ロボット制御などの実応用において深刻な問題となります。しかし、ワールドモデルがいつ、なぜ幻覚を起こすのか、そしてそれをどのように防ぐのかについては、十分に理解されていませんでした。 ## 2. 提案手法のアプローチ：どのようなアイデア・アプローチで問題を解決しようとしたか 本研究では、ワールドモデルにおける幻覚が、主に「データカバレッジの欠如」、すなわちモデルが学習時に十分に観測していない状態・行動空間の領域に集中しているという仮説を立てました。この仮説を検証し、幻覚を予測・防止するために、以下の3つのアプローチを取ります。 1. **大規模データセットの構築とモデルの訓練:** 427時間、210タスクに及ぶ、視覚的ワールドモデリングのための大規模データセット「MMBench2」を構築しました。このデータセットには、真の行動、報酬、そしてライブシミュレーターが含まれています。このデータセットを用いて、3億5千万パラメータのワールドモデルを訓練しました。 2. **幻覚の分類と予測指標の開発:** 訓練したモデルを分析し、幻覚を3つの異なるモード（知覚、行動周辺化、シーン分岐）に分類しました。さらに、これらの幻覚がモデルのどの段階で発生するかを特定し、モデル内部の信号を利用して幻覚を予測する3つの指標（トークナイザー往復残差、フロー不安定性、シード間分散）を開発しました。 3. **幻覚の予測・緩和策の実施:** * **学習時のカバレッジギャップの解消:** カバレッジを意識したサンプリング手法を開発し、学習データセットにおける低カバレッジ領域の重み付けを調整することで、訓練時の幻覚を低減させました。 * **オンラインでのカバレッジギャップの解消:** 開発した幻覚予測指標を「好奇心報酬」として利用し、モデルが幻覚を起こしやすい領域をターゲットにしたデータ収集をオンラインで実施しました。これにより、データ効率の良いファインチューニングが可能となり、未知の環境にも少数の実環境軌道で適応できるようになりました。 ## 3. 成果：提案手法により、何が達成できたのか 本研究は、ワールドモデルにおける幻覚がデータカバレッジの問題に起因することを実証し、それを予測・緩和する効果的な手法を開発しました。具体的には、以下の成果を達成しました。 * **幻覚の予測:** 開発した3つの予測指標（トークナイザー往復残差、フロー不安定性、シード間分散）が、ラベルなしで、かつ追加学習なしに、モデルの幻覚発生を高い精度で予測できることを示しました。これらの指標は、モデルのパフォーマンスと強く相関し、幻覚イベントを正確に追跡できることが確認されました。 * **幻覚の緩和:** * **学習時のカバレッジ向上:** カバレッジを意識したデータサンプリング手法を導入することで、学習データセットにおける低カバレッジ領域の重み付けを調整し、3つの幻覚モードすべてを同時に低減させることに成功しました。 * **データ効率の良い適応:** 幻覚予測指標を好奇心報酬として利用したターゲットデータ収集により、訓練済みのワールドモデルを、わずか50個の実際の環境軌道で、全く未知の環境に適応させることが可能になりました。これは、人間によるデータ収集と同等の効果に匹敵するデータ効率の良さを示しています。 * **幻覚の分類:** 幻覚を「知覚」「行動周辺化」「シーン分岐」の3つのモードに分類し、それぞれがモデルパイプラインの異なる段階（エンコーダー・デコーダー、行動条件付け、マルチステップロールアウト）に紐づいていることを明らかにしました。 * **大規模データセットの提供:** MMBench2という、視覚的ワールドモデリングのための大規模で多様なデータセットを開発・公開しました。これは、将来の研究におけるベンチマークとして貢献します。 ## 4. 課題：本手法の限界・課題にはなにか。本文中で著者が言及しているものと、あなたが想定するもの ### 本文中で著者が言及している課題： * **大規模モデルへの適用:** 本研究の知見が、数十億パラメータを持つようなさらに大規模なモデルや、センサーノイズや部分的な観測を伴う実際のロボットデータにどの程度適用できるかは、経験的な問題として残っています。 * **計算コスト:** 大規模なワールドモデルの訓練には significant な計算リソースが必要であり、そのコストは依然として課題です。 * **データセットの多様性:** より多様なデータセットの必要性が示唆されています。 ### あなたが想定する課題： * **予測指標の汎用性:** 開発された幻覚予測指標が、MMBench2のデータセットやDreamer4のような特定のモデルアーキテクチャに強く依存している場合、他のアーキテクチャや異なるドメインのワールドモデルに対してどこまで汎用性があるかは不明です。 * **「幻覚」の定義の曖昧さ:** 本論文では幻覚を「現実から乖離した、流暢で視覚的にあり得る出力」と定義していますが、その境界線は曖昧であり、特に低カバレッジ領域での「妥当な外挿」と「望ましくない幻覚」の区別が難しい場合があります。 * **リアルタイム性の制約:** 幻覚予測指標の計算や、それを用いたオンラインデータ収集・ファインチューニングは、リアルタイムの意思決定を必要とするアプリケーションにおいては、計算オーバーヘッドが問題となる可能性があります。 * **カバレッジとパフォーマンスのトレードオフ:** データカバレッジを向上させるためのサンプリング手法が、必ずしも全てのタスクや将来の予測タスクにおいて最適なパフォーマンスを保証するわけではない可能性があります。特定のタスクに特化したファインチューニングが別途必要になる場合も考えられます。 ## 5. 技術詳細：提案手法の技術的な詳細について、できるだけ詳しく。数式やアルゴリズム、モデル構造なども可能な限り詳しく ### モデルアーキテクチャ： 本研究では、Dreamer4のアーキテクチャをベースにした、3億5千万パラメータの行動条件付きワールドモデルを使用しています。このモデルは、以下の2つの主要コンポーネントから構成されます。 1. **トークナイザー（Tokenizer）:** * **構造:** 対称的なエンコーダー・デコーダーTransformerアーキテクチャ。 * **エンコーダー:** * 入力: 画像観測（例: 64x64 RGB） * 処理: 入力画像をパッチに分割し、学習可能な潜在クエリ（learnable latent queries）とパッチクエリ（patch queries）を用いて、自己注意機構（self-attention）により空間情報をエンコードします。 * 出力: 空間情報を潜在コード（latent codes）に圧縮します。 * パラメータ数: 50M * **デコーダー:** * 入力: エンコーダーから得られた潜在コード * 処理: 潜在コードを条件として、画像（元の観測）を再構築します。 * 訓練目的: マスク化再構築（masked reconstruction）。入力パッチの一部をマスクトークンに置き換え、マスクされた位置のみでピクセルMSEとLPIPSの損失を計算します。 * **空間パッキング/アンパッキング:** * トークナイザーは、フレームごとに `(n_spatial, d_spatial) = (32, 128)` の次元を持つ潜在トークンを生成します。 * エンコーダー出力: `(H/8, W/8, d_model)` -> `(N_patches, d_model)` -> `(N_queries, d_queries)` * デコーダー入力: `(N_queries, d_queries)` -> `(N_patches, d_model)` -> `(H/8, W/8, d_model)` 2. **ダイナミクスモデル（Dynamics Model）:** * **構造:** 2億5千万パラメータのブロック因果Transformer（block-causal Transformer）。 * **入力:** トークナイザーによって生成された、パッキングされた潜在トークン、行動トークン、およびその他の条件トークン。 * トークンレイアウト（Timestepごと）: `[ACTION x 1, SHORTCUT x 1, SPATIAL x 32, REGISTER x 4, AGENT x 4]` * `ACTION`: `(d_action_dim)` 次元。パディングされた行動ベクトル。 * `SHORTCUT`: ノイズレベルの離散化された埋め込み。 * `SPATIAL`: `32` 個の潜在トークン。 * `REGISTER`: `4` 個の学習可能な「シンク」トークン。 * `AGENT`: `4` 個のトークン。報酬予測や行動クローニングヘッドに使用。 * **訓練目的:** ショートカットフローマッチング（shortcut flow-matching）。 empirical one-step regression と self-consistency bootstrap の組み合わせ。 * **Empirical One-step Regression:** 実際の次の状態への遷移を学習。 * **Self-Consistency Bootstrap:** より粗いタイムステップ（例: 2ステップ先）の予測を、より細かいタイムステップ（1ステップ先）のターゲットに蒸留。 * **推論:** ショートカットEulerインテグレーター（step size `dt_inference = 1/32`）。 * **コンテキスト破損（Context Corruption）:** 推論時に過去のトークンにノイズを適用。 ### 幻覚の3つのモード： 1. **知覚的幻覚（Perceptual Hallucination）:** * **発生段階:** トークナイザー（エンコーダー・デコーダーペア）。 * **内容:** モデルが訓練データセットで見たことのないシーン構造（例: 見たことのない迷路のレイアウト）を、学習した潜在空間の「語彙」内で最も近いインディストリビューション（in-distribution）の例に射影して再構築してしまう。例えば、壁の配置は異なるが、エージェントと目標の位置は正しいといった場合。 * **影響:** この破損したシーンが「真の」状態としてダイナミクスモデルに与えられ、その後のロールアウトに影響する。 2. **行動周辺化（Action-Marginalized）幻覚:** * **発生段階:** ダイナミクスモデルの行動条件付け。 * **内容:** 与えられたコンテキスト（過去の状態と行動）に対して、予測される次の潜在状態が、入力された行動にほとんど依存しない。結果として、モデルは特定の行動を取っても、あたかも動画ジェネレーターのように、行動とは無関係な（行動周辺化された）未来へ遷移してしまう。 * **検出方法:** バッチ内の行動をランダムにシャッフルした際のフローMSEの変化を測定することで検出。幻覚を起こすモデルは、行動の変更に対してMSEの変化が小さい。 3. **シーン分岐（Scene-Diverging）幻覚:** * **発生段階:** マルチステップロールアウト。 * **内容:** オートリグレッシブなロールアウトにおいて、予測ホライゾンが増加するにつれて誤差が累積し、物理的にあり得ないイベント（例: ポンゲームでボールがゴールに入った後に突然プレイに戻る）が発生する。これは、特にデータカバレッジが低い状態（low data coverage regions）で頻繁に発生する。 ### 幻覚予測指標： これら3つの指標は、モデル内部の信号を利用し、ラベルなしで幻覚を検出するために開発されました。 1. **トークナイザー往復残差（Tokenizer Round-trip Residual）:** * **目的:** 知覚的幻覚を検出。 * **計算:** `u_r = || \hat{z} - Encode(Decode(\hat{z})) ||` * `\hat{z}`: ダイナミクスモデルによって予測された次の潜在状態。 * `Encode()`、`Decode()`: トークナイザーのエンコーダーおよびデコーダー。 * **解釈:** 予測された潜在状態をデコードして画像にし、それを再度エンコードした際に生じる残差の大きさを測る。この残差が大きい場合、デコードされた画像がトークナイザーの学習した多様体（manifold）から外れており（例: 破損したシーンレイアウト、偽のオブジェクト）、知覚的幻覚が発生している可能性が高い。 2. **フロー不安定性（Flow Instability）:** * **目的:** 行動周辺化幻覚を検出。 * **計算:** ダイナミクスモデルの推論時に、ショートカットEulerインテグレーターの複数サブステップ間での潜在状態の変化の安定性を評価。具体的には、後半のサブステップにおけるデノイザーのクリーンターゲット予測が、 successive Euler integration substeps の間でどれだけ変化するかを平均する。 * **解釈:** 健全で条件付けの良いダイナミクスヘッドは、安定した最終状態に素早く収束する（低い `u_f`）。一方、行動条件付けの信号が弱いヘッドは、サブステップ間で振動し続け、不安定になる（高い `u_f`）。 3. **シード間分散（Inter-seed Variance）:** * **目的:** シーン分岐幻覚を検出。 * **計算:** 固定された過去の状態と行動に対して、独立したノイズシードを用いた複数のデノイジング（ロールアウト）軌道を生成し、それらの軌道間の差異を測定する。 * **解釈:** シード間で予測が大きく分散する領域は、まさにマルチステップロールアウトが分岐しやすい（uncertain）領域である。これは、データカバレッジが低い状態では、モデルが複数の可能性のある（そしてしばしば誤った）未来に分岐しやすいことを反映している。 **動的な正規化（Dynamism Normalization）:** これらの予測指標は、シーンの活動（動き）に影響されるため、動的な正規化を適用します。 * `u_r_norm = u_r / dynamism` * `u_f_norm = u_f / dynamism` * `u_s_norm = u_s / dynamism` ここで `dynamism` は、潜在表現のステップごとのRMS変化として計算されます。これにより、各予測指標は、シーンの動きと比較したモデルの不確実性を追跡します。 ### カバレッジを意識したデータ収集・学習： 1. **カバレッジを意識したサンプリング（Coverage-Aware Sampling）:** * **目的:** 学習時のデータカバレッジギャップを埋める。 * **手法:** 既存のデータセットをリサンプリングし、状態・行動空間における過小表現領域（under-represented regions）の重み付けを増やす。 * **例:** MMBench2の訓練データセット全体で、タスクごとのサンプリングを均等化する。 2. **好奇心報酬としての幻覚予測指標:** * **目的:** オンラインでデータカバレッジギャップを埋める。 * **手法:** ライブ環境とのインタラクション中に、ワールドモデルで候補軌道をロールアウトし、開発した幻覚予測指標（正規化後）でスコアリングします。最も高いスコア（＝最も幻覚を起こしやすいと予測される）の軌道を実行し、そのデータを収集します。 * **効果:** この方法で収集されたデータは、モデルが以前に幻覚を起こした遷移をカバーするように設計されており、データ効率の良いファインチューニングにつながります。 ### MMBench2 データセット： * **規模:** 427時間、210タスク、約20Mフレーム（訓練用）、高解像度。 * **内容:** 視覚的ワールドモデリングに特化。 * 真の行動と報酬ラベル * ライブシミュレーター * 多様な行動（専門家デモンストレーション、人間プレイ、ランダム、ノイズ付加、行動変換など） * 多様なタスクドメイン（ロボット操作、運動、ナビゲーション、アーケードゲーム、古典制御など） * **特徴:** MMBenchを基盤としつつ、行動の多様性を大幅に増加させ、人間によるプレイデータも収集。 ### 訓練フェーズ: 1. **ベースライン訓練:** MMBench2の訓練コーパス全体で、トークナイザーとダイナミクスモデルを事前訓練。 2. **カバレッジを意識したミッドトレーニング:** 事前訓練に加えて、カバレッジを意識したサンプリング手法を適用。 3. **ファインチューニング:** * **同一タスク:** 事前訓練済みのモデルを、カバレッジを意識したサンプリングでさらに訓練。 * **未見タスク:** ターゲットタスクのデータ（50軌道程度）を用いて、好奇心報酬（幻覚予測指標）を基にしたターゲットデータ収集とファインチューニングを実施。 ## 6. この論文を140字以内のツイートで要約すると？ ワールドモデルの「幻覚」はデータ不足が原因！🔑
